# 期次成果表与期报

每次外业完成、平差完毕后，向业主提交"期成果"。期成果是后续总结报告、年度报告、专家评审的基础数据来源。

## 1 期成果体系

| 报告类型 | 频率 | 报送对象 |
| --- | --- | --- |
| **监测日报** | 每天 / 当班 | 业主、监理、总包、相关方 |
| **监测周报** | 每周 | 业主、监理 |
| **监测月报** | 每月 | 业主、监理、上级主管 |
| **期成果表（期报）** | 每监测期次 | 业主、监理、专家组 |
| **预警快报** | 异常 4 h 内 | 业主、监理、应急组 |
| **阶段性报告 / 总结报告 / 年度报告** | 见 `summary-report.md` | 业主、专家评审 |

## 2 监测日报内容

1. 当日观测范围（区间、断面、点号清单）
2. 仪器使用与 i 角检校结果
3. 外业完成度（计划 vs 实际）
4. 当日数据初判：是否有超限/异常
5. 巡视巡查（裂缝、渗漏、错台、附属设施异常）
6. 安全/作业情况、天窗点利用
7. 次日计划

## 3 监测周报 / 月报内容

- 期内总观测期次与覆盖率
- 累计变化量分布（按区间统计最大值、平均值、超阈点数）
- 重点关注断面/点位的曲线
- 与上一周/月趋势对比
- 控制网/工作基点稳定性结论
- 巡视巡查汇总（位置、现象、照片）
- 异常处置与闭环情况
- 下周/下月计划

## 4 期次成果表（核心交付）

### 4.1 文件构成
| 文件 | 用途 |
| --- | --- |
| `<期次号>-沉降成果表.xlsx/pdf` | 道床/结构/联络通道/道岔/建构筑物分表 |
| `<期次号>-收敛成果表.xlsx/pdf` | 内径收敛、净空断面 |
| `<期次号>-水平位移成果表.xlsx/pdf` | 视项目 |
| `<期次号>-控制网联测报告.pdf` | 见 `baseline-network.md` |
| `<期次号>-平差报告.pdf` | 见 `data-processing.md` |
| `<期次号>-期监测报告.pdf` | 文字说明、结论、附图 |
| 原始记录 | 数字水准 .dat、全站仪原始、扫描点云 |

### 4.2 命名建议
`<项目编号>-<期次年月>-<监测类别>-<上下行>` 例：`XXX-2024-08-沉降-上行.xlsx`。

### 4.3 成果表列结构
参见 `data-processing.md` §6。

### 4.4 抬头必含项
- 项目名称
- 期次号 / 观测日期 / 签发日期
- 监测对象、监测频率、监测等级
- 高程/平面基准、正负号约定
- 仪器型号、检定证书编号
- 监测单位、监理单位、主测/内审/外审签名

## 5 期监测报告章节模板（建议 8 节）

1. **概述**
   - 项目背景、监测目的、本期范围与频率依据
2. **本期工作量**
   - 区间、断面、点号、外业天数、仪器使用情况
3. **基准与控制网**
   - 高程/平面基准、深桩点检测、工作基点稳定性判定结论
4. **数据处理**
   - 平差软件与版本、平差结果概要、精度评定（先验/后验单位权中误差、最弱点）
5. **监测成果分析**
   - 分类别（沉降、收敛、水平位移）给出最大值、平均值、超阈点数与位置
   - 与上一期对比趋势
   - 关键断面/点位曲线
6. **巡视巡查**
   - 现场异常（裂缝、渗漏、错台、施工干扰）
7. **结论与建议**
   - 整体安全判定
   - 是否进入预警/加密、改正与处置
8. **附录**
   - 成果表、平差报告、控制网联测报告、巡视照片、变更记录

## 6 报告内容控制规则（务必遵守）

- 不写"无变化"——所有变化量都需给出数值与限值比较
- 不写"基本稳定"——必须说明判定依据（限差、中误差倍数）
- 任何"重测、改算、补测"必须在报告中显式列出
- 工后变形量必须给出"工后零点期次"
- 成果表与文字描述/图表数值必须一致；不一致即视为不合格

## 7 期次切换的衔接

- 上期归档完成后才能开新期
- 上期发现的预警/超阈点必须在本期开篇说明处置进展
- 测点重测/迁移必须保留原点号映射表
- 平差软件升级、仪器更换必须在期报"基准与控制网"章节说明

## 8 常见 Pitfall

- 期次号与观测日期不一致（外业 12 月底、报告 1 月签发，期次写成下期）
- 成果表与曲线图变化量符号不一致
- 巡视巡查内容缺位置坐标，仅"某段隧道"
- 工作基点稳定性结论复制上期，未做本期判定
- 加密观测期把"加密期"数据与"常规期"数据合并出表，导致曲线密度异常
- 报告页脚未签名/未盖章直接发送
