# 归档与交付

按"地铁保护监测内业资料归档目录"通用清单组织归档，确保任何期次、任何时刻可被业主、监理、第三方核查追溯。

## 2 文件命名规范

通用：`<项目编号>_<期次>_<类别>_<上下行/范围>_<版本>.<扩展>`

示例：
- `XXX_2024-08-12_沉降_上行_V1.0.xlsx`
- `XXX_2024-08-12_平差报告_控制网_V1.0.pdf`
- `XXX_2024-08-12_预警快报_第03号.pdf`

**禁止**使用中文路径名（特别是与业主/第三方交换时）；可以在文件名 _ 之后增加中文标签辅助识别。

## 3 关键交付物清单

| 时点 | 必交付物 |
| --- | --- |
| **项目启动** | 公司资质、监测方案、仪器报审单、技术交底记录 |
| **初始值采集** | 测点埋设记录、验收记录、首期控制网联测报告、初始值报告（含 3 次独立观测）|
| **每期次完成** | 期监测报告、成果表（沉降/收敛/水平位移）、平差报告、控制网联测（含稳定性）、巡视报表、原始记录 |
| **每月** | 月报、外业人天与频率执行台账、仪器维护与 i 角检验记录 |
| **每年** | 年度总结报告、检定证书更新、变更台账 |
| **合同期末** | 总结报告、全套档案移交清单、专家评审意见与回复 |
| **异常期** | 预警快报正本/回执、加密观测记录、消警备案 |

## 4 原始数据保存

| 数据类型 | 格式 | 保存策略 |
| --- | --- | --- |
| 数字水准 | .dat、.gsi、.lev | 原始 + 平差用副本 |
| 全站仪 | .gsi、.raw、.txt | 原始 + 处理副本 |
| 三维激光 | .fls、.e57、.rcs、.las | 原始站点云 + 拼接成果 |
| 平差报告 | .pdf + 工程文件（.csg/.adj） | 工程文件可复现 |
| 巡视照片 | .jpg + 元数据 | EXIF 含 GPS / 时间 |
| 成果表 | .xlsx + .pdf | 双格式 |
| 报告 | .docx + .pdf | 双格式 |

**保留期**：合同要求不低于 5 年；运营本体监测建议**永久保存**（含云端备份 + 线下移动硬盘双备份）。

## 5 数据完整性自检清单

每期归档前由内业责任人逐项打钩：

- [ ] 原始数据全部入库（按测段/区间齐全）
- [ ] i 角检验记录入档
- [ ] 控制网联测报告完成、稳定性判定明确
- [ ] 平差报告先验/后验单位权中误差、最弱点齐全
- [ ] 成果表抬头基准、正负号、签名齐全
- [ ] 文字报告与成果表数值一致
- [ ] 巡视报表含照片与位置坐标
- [ ] 异常事件（含上期遗留）有闭环说明
- [ ] 业主/监理签收回执（如已报送）
- [ ] 文件命名符合规范

## 6 移交档案要求

合同末期或委托终止时，向业主完整移交：

1. **纸质档案**：装订成卷（每卷有目录、卷内目录、页码、检索表）
2. **电子档案**：移动硬盘 + 云端双份；按§1 目录骨架整理；附 SHA-256 校验文件
3. **移交清单**：双方签认；保留盖章原件至少 5 年
4. **关键数据库**（如使用过专用系统）：导出标准格式（CSV/Excel）一份

## 7 与上级数据系统的衔接

如运营单位有"工程档案系统"或"监测数据平台"，按其接收规范上传：
- 命名规则一致
- 元数据完整（点号、坐标、期次、平差版本）
- 校验日志保留

> 本技能不负责设计此类平台，仅负责输出符合规范的数据交付物。

## 8 常见 Pitfall

- 原始数据未及时入库（外业当天没导出），事后追溯困难
- 平差工程文件丢失，只保留 PDF 报告，专家无法回算
- 三维激光只存拼接成果，未保留原始站点云
- 巡视照片无 GPS/位置信息
- 文件命名使用中文路径，跨系统传输乱码
- 预警快报未保留业主签收回执
- 移交档案缺少 SHA-256 等校验，遭质疑数据被改

## 9 单期归档操作小抄

```
1) 当日外业结束 → 原始数据导出到 07_原始记录/<期次>/
2) 当日内业 → i 角检验入 12.4；巡视入 06；日报入 05
3) 完成本期所有外业 → 控制网平差 → 10_控制网联测/<期次>/
4) 监测点平差 → 成果表 → 11.4_期监测报告/<期次>/
5) 期报内审 → 外审 → 签发 → 上传 + 报业主
6) 预警事项 → 13.3_预警与消警记录/<期次>/
7) 月末汇总 → 09_监测月报/
8) 年末归档 → 11.2_年度报告/
```
