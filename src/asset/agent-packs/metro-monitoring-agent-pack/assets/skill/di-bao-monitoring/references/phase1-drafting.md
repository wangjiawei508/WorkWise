# 1 期：初稿起草工作流

本 reference 解决"**从零到一**"的问题：拿到资料后，怎么从空白模板写到送内审的状态。

## 一、不要从第 1 章开始顺写

新手最常犯的错误，是打开模板从"1.1 工程概述"开始顺序往下写。结果往往是：

- 写到第 3 章发现监测范围还没定 → 回去改第 1 章的"概述"
- 写到第 5 章发现测点数量算不准 → 回去改第 3 章的"工作量"
- 写到第 11 章发现报警值阶段划分跟第 3 章频率表对不上 → 全文返工

**正确顺序**是先**锚定关键参数**（监测范围、阶段划分、报警值），再回填描述性章节。

## 二、推荐起草顺序（共 7 步）

### Step 1：先定锚——监测范围 + 监测阶段

打开 `assets/plan-template.md`，先填两节，其他章节先空着：

#### (1) 章节 3.1 监测范围

依据：**安评报告 + 工程类型公式**（详见 `technical-standards.md` 第 6 节）。

写法骨架：
```markdown
本项目 {工程类型} 正投影范围约 {XX} m，
按照"投影 + 两侧 {3H / 30 环 / 1.5 倍埋深}"原则外延，
故监测范围总长 {XX} m，对应 {站名}~{站名} 区间
上行线 {环号}~{环号}，下行线 {环号}~{环号}。
其中 {重点区环号} 为重点监测区域，
在原 5 环 1 个断面的基础上加密 1 个断面，
上、下行线共计加密 N×2 = 2N 个断面。
```

#### (2) 章节 3.4 监测阶段划分

**从工筹横道图反推**——这是初稿阶段的核心动作：

1. 在工筹图上**用红笔圈出对轨道交通有扰动的工序**（开挖、顶进、打桩、拆撑等）
2. 把这些工序按**对既有结构的扰动等级**归类：
   - **扰动小**：准备、围护施工、回填、绿化恢复 → 归为"常规阶段"
   - **扰动中**：基坑开挖（部分深度）、土方运输、顶管出洞 → 归为"敏感阶段"
   - **扰动大**：开挖至坑底、换撑、拆撑、顶进至轴线、注浆置换、嵌缝、钢板桩打拔、盾构穿越 → 归为"关键阶段"
3. 阶段数量**通常 3–5 个**（多了评审会问，少了关键工序不突出）。代表样例：
   - 6 号线市中医院方案：3 阶段（主体基坑施工 / 望-天盾构施工 / 工后监测）
   - 段塘大桥方案：5 阶段（围堰施工 / 桩基施工 / 桥下通道开挖 / 老桥拆除 / 工后监测）
   - 1 号线 A5 顶管方案：3 阶段（顶管准备 / 顶进 / 后期注浆与工后）

阶段一旦定下来，**整份方案的频率表、报警值表、附图、工作量都按这个阶段来组织**。

### Step 2：填充施工筹划（章节 1.5）

把 Step 1 圈过的工筹图整理成横道表，按"分部工程 / 起止日期 / 工期 / 备注"四列写入章节 1.5。

**注明工筹版本号 / 日期**，例如 "依据《XX 工程施工筹划（2025-09-10 版）》"——后期甲方改工筹时，你能据此主张"重大变更"。

### Step 3：监测点布设 + 工作量预估（章节 5 + 3.5）

依次：

1. **断面数计算**（用 `scoping-and-pricing.md` 公式）
2. **每断面测点配置**（沉降+水平位移+收敛，按工况勾选）
3. **总测点数 = 断面数 × 每断面测点数**
4. **监测次数 = 阶段时长 × 频率 × 阶段权重**
5. **填入章节 3.5 预估工作量表**

工作量数字必须和 6 章监测方法、12 章设备 / 人员清单**对得上**。

### Step 4：监测重难点与应对措施（章节 2.3）⚠️ 最重要

这是评审最敏感、初学者最容易写废的章节。**单独看 `references/key-difficulties.md`**，按"诊断→推演→生成"三步工作流，**不要直接抄条目库**。

### Step 5：报警值与预消警（章节 11）

**别照抄标准报警值表**——这一步要做"项目化裁剪"：

1. 翻安评报告，找"变形预测值"章节
2. 把安评的预测值 × 0.6 = 预警值；× 0.8 = 报警值；× 1.0 = 控制值（粗规则，需复核）
3. 与标准报警值（±5/±7/±10mm）取**严者**作为本项目控制值
4. **分阶段细化**：给每个施工阶段列单独一行（这一步会获评审加分）
5. 预消警流程**照抄模板**——这是程序性章节，运营单位有标准流程图

### Step 6：回填描述性章节（章节 1.1–1.4、第 4 章、第 6–10 章、第 12–14 章）

到这一步，方案骨架已经立住，再补描述性内容：

| 章节 | 起草要点 | 复用程度 |
|---|---|---|
| 1.1 工程概述 | 从施工组织设计抄要点，重写不超过 300 字 | 不可复用 |
| 1.2 轨道交通概况 | 从公司历史方案抄同区间内容 | 高度复用 |
| 1.3 地质 | 勘察报告关键参数 + 一张地层表 | 不可复用 |
| 1.4 周边环境 | **必须明确"重车停放/行走位置"** | 不可复用 |
| 1.6 监测等级 | 一句话引用分级文件 | 高度复用 |
| 1.7 既有沉降数据 | 找运营分公司或自家历史数据 | 不可复用 |
| 1.8 安评 | 摘安评结论 5 行 | 不可复用 |
| 第 4 章 控制网 | **同一城市的方案高度复用**，只改环号 | 高度复用 |
| 第 6 章 监测方法 | 全公司统一文风 | 高度复用 |
| 第 7-10 章 巡视、信息反馈、数据保密、轨行区 | 完全程序性 | 几乎全复用 |
| 第 12 章 人员设备 | 改人员名单和设备清单 | 半复用 |
| 第 13-14 章 保障、配合 | 全公司统一 | 完全复用 |

> **复用纪律**：高度复用章节也必须做"两轮替换"——线路名、车站名、环号、日期，至少替换两遍并通读一遍。批量替换出错也是常见低级失误。

### Step 7：附图清单与组卷

**初稿阶段附图先用文字描述**，不强行画图。在每一节插入：

```
【图 X.X-N 图名】（待补：从 XXX 文件提取，定稿前完成）
```

定稿前再请同事或自己集中绘图，避免起草过程中反复修改图纸。

## 三、每节"无中生有"的写法

初稿阶段最难的是：**资料里没现成段落，要自己组织语言**。下面给几个高频章节的"生成公式"：

### 章节 1.4 周边环境（典型"自己组织"段落）

```
公式：现场踏勘 + 卫星图 + 红线图

写作骨架：
本项目监测范围内主要周边环境如下：
- 北侧：{距离 X m} {环境物}（{补充说明，如"含主要交通干道，重载车辆通行频繁"}）
- 南侧：……
- 东侧：……
- 西侧：……
重载车辆停放位置位于 {方位}，距离 X 号线投影最近 {X m}，行走路线为 {描述}。
```

### 章节 1.7 既有沉降数据

```
公式：找运营单位要数据 → 取近 1-3 年沉降曲线 → 三句话总结

写作骨架：
根据运营分公司 {年份} 月度监测报告，X 号线 {站}~{站} 区间
近 {N} 年累计沉降最大值 {±X mm}（位置：上行线 {环号}），
近 6 个月平均沉降速率 {X mm/月}，
工后变形已趋于稳定。详细数据见附件 X。
```

### 章节 7.1 巡视巡查内容

```
公式：照抄模板 + 加 1-2 条项目特定项

通用条目（照抄）：
- 隧道结构外观（裂缝、错台、渗漏）
- 道床状态（破损、裂缝）
- 设备安装情况

项目特定条目（必加 1-2 条）：
- {基坑工程}：周边地表沉降、围护结构变形目视检查
- {顶管工程}：顶管机姿态、出土量异常
- {桥梁迁建}：钢板桩打拔同步观察、桥桩施工振动
```

## 四、初稿完工自查（送内审前 30 分钟）

按下面 10 条快速自查，不达标就别送内审：

1. [ ] 目录与正文章节编号、标题字字对齐
2. [ ] 监测阶段划分与施工筹划日期能对上
3. [ ] 测点总数 = 断面数 × 每断面测点数（手算复核一遍）
4. [ ] 工作量表的总监测次数 = ∑（阶段时长 × 频率）
5. [ ] 报警值表三栏（预警/报警/控制）齐全且分阶段
6. [ ] 重难点章节有 5-8 条且对应 5-8 条应对措施
7. [ ] 安评报告的风险等级、变形预测值在方案中出现并被引用
8. [ ] 工程总平面图位置已留出（即使是占位）
9. [ ] 应急联络名单含 24h 电话
10. [ ] 全文搜索 `{` 和 `待补`，确保占位符全部清除（最容易踩雷）

## 五、版本管理建议

- `v0.1-初稿`：完成所有章节，可能有 `【待补】` 标记
- `v0.2-内审稿`：清除所有占位符，自查 10 条通过，发部门内审
- `v0.3-内审修订`：内审纪要逐条回填
- `v0.4-送审稿`：送上级或运营单位审查
- `v0.5-专审修订`：专家评审后修改
- `v1.0-定稿`：所有签章页齐全

每次出版本，文件名含日期：`方案-v0.2-20251015.md`。
