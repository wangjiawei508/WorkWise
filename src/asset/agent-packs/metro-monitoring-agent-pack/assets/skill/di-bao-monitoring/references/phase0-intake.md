# 0 期：资料收集与立项决策

新接到一个项目时，**别急着打开模板填空**。先用这份 reference 把资料、决策、报价三件事捋清楚。这一步省事，后面所有阶段都顺。

## 一、资料索取清单（一次性问全）

下表是从甲方/施工方/设计院索要资料的**完整清单**。**强烈建议把这份表整张发给对方**，让对方一次性给齐，避免后面反复要——这是初稿阶段最大的时间杀手。

### 模板邮件/微信话术

```
{X 总}您好，

为编制{工程全名}施工期间轨道交通 X 号线控制保护区监测方案，
烦请提供以下资料（按必须 / 建议两档分级）：

【必须项 / 没有就没法动笔】
1. 工程总平面图（CAD 或高清 PDF，含与既有轨道交通的位置关系）
2. 施工组织设计 + 各阶段施工方案
3. 施工筹划横道图（含具体日期）
4. 工程地质勘察报告（含水文地质）
5. 围护设计图（基坑/顶管/管廊工程必备）
6. 《对轨道交通结构安全影响评估报告》（即"安评报告"）
7. 监测合同 / 招投标技术规格书
8. 轨道交通线路平面图 / 区间结构图 / 对应环号信息

【建议项 / 有则更准】
9. 既有线工后沉降近 1-3 年监测数据
10. 既有线结构加固历史资料（如果有过盾构加固、桩位等）
11. 与本工程交叉/邻近的其他建设项目情况
12. 上一版（若有）方案及评审意见

资料齐了我这边大约 X 个工作日出初稿，方便的话本周内发过来？
```

### 资料完整度自检表

| 必须项 | 用途 | 缺失影响 |
|---|---|---|
| 工程总平面图 | 章节 1.1、1.4、附图 1 | ❌ 无法画位置关系图，评审 100% 打回 |
| 施工组织设计 + 各阶段方案 | 章节 1.5、3.4 | ❌ 无法划分监测阶段 |
| 施工筹划横道图 | 章节 1.5 | ❌ 监测周期无法定 |
| 勘察报告 | 章节 1.3 | ❌ 无地质参数表 |
| 围护设计图 | 基坑章节 | ⚠️ 可暂以方案描述替代 |
| **安评报告** | 章节 1.8、3.1、11.1 | ❌ **最致命**——无法定监测范围、报警值；评审必问 |
| 监测合同 / 技规 | 章节 2、报价 | ⚠️ 可暂用框架协议 |
| 轨道线路图 / 环号 | 章节 1.2、5.1 | ❌ 无法定测点位置 |

> **铁律**：**安评报告**没拿到之前**不要动笔写监测范围和报警值章节**。这两节的所有数据都是从安评里来的，照抄会被评审一眼看穿。

### 高架项目补充清单（受影响轨道为高架区间/出入场线/车辆段引桥时叠加）

```
【高架线路补充资料 / 必须项】
H1. 桥梁设计总图（含墩号、桥跨、墩高、桥型）
H2. 桥墩明细表（墩号 / 墩高 / 墩柱形式 / 承台与桩长）
H3. 桥面铺装与道床状态调查表（**主桥/辅桥分别**注明是否铺道床）
H4. 主辅桥关系图（共线 / 独立 / 拼宽）
H5. 桥梁近 1-3 年养护检测资料（含外观普查、墩柱倾斜历史值）
H6. 桥下既有道路 / 河道 / 管线断面图
H7. 接触网平面与本项目工期内的停电计划
H8. 桥梁专项的对轨道交通结构安全影响评估报告
```

高架项目还须前置调查 4 件事，否则**评审必问、必打回**：

| 调查项 | 不调查的代价 | 调查方法 |
|---|---|---|
| **辅桥是否铺设道床** | 评审打回（真实审查意见原件） | 现场踏勘 + 设计院书面回复 |
| **桥墩编号体系** | 测点统计表无法编排 | 设计院出图 + 现场核对 |
| **接触网热滑时间** | 自动化撤场冲突 | 运营调度处书面确认 |
| **风险等级** | 频率与报警值定不下来（真实审查意见） | 安评 + 业主签认 |

详 `elevated-line.md` §二。

### 资料延迟到位的应对

如果对方一直拖某项资料，分两种处理：

- **必须项延迟** → 暂停起草，邮件 + 微信双通道催，并抄送项目负责人；同时**写一份"待资料到位"占位章节**，等齐了再填
- **建议项延迟** → 正常起草，相关位置加 `【待补：XXX 资料到位后补充】` 醒目标记，定稿前清除

## 二、监测等级判定

监测等级直接决定后面所有参数——精度、频率、报警值都跟着变。按下面三步走：

### Step 1：风险等级（由安评报告给出）

按《XX 市轨道交通控制保护区和规划控制范围分级审查管理办法》分为 **一/二/三级**：

| 风险等级 | 一般描述 | 典型工况 |
|---|---|---|
| **一级** | 高风险，工程对运营影响巨大 | 盾构下穿运营隧道（净距 < 3 m）、深基坑紧邻 |
| **二级** | 中等风险 | 顶管邻近、基坑邻近、桥桩施工 |
| **三级** | 较低风险 | 邻近建筑、远端管廊 |

### Step 2：监测等级（由风险等级映射）

通常监测等级 = 风险等级。但若安评建议提高或工程合同明确，则按合同/安评。

### Step 3：监测等级 → 参数表（直接套）

| 参数 | 一级 | 二级 | 三级 |
|---|---|---|---|
| **沉降精度** | ±0.3 mm/km | ±0.5 mm/km | ±1.0 mm/km |
| **自动化频率** | 12 次/天起步 | 6 次/天 | 4 次/天（部分场景） |
| **人工复核频率** | 1 次/3 天 | 1 次/7 天 | 1 次/10 天 |
| **预警/报警/控制（沉降）** | ±3 / ±5 / ±7 mm | ±5 / ±7 / ±10 mm | ±7 / ±10 / ±15 mm |
| **报送要求** | 日报 + 警情快报 | 日报 + 周报 | 周报 + 月报 |

> 默认起草按 **二级** 设置；一级要在 SKILL.md 主流程里特别标注"加严"，三级要在合同 / 安评佐证下才能调宽。

## 三、模板选择决策

回答 3 个问题：

1. 既有线是否已运营？
   - 是 → 必走 **模板 A**
   - 否 → 进入问题 2

2. 招标 / 委托方是否要求按"地保监测监护标准化手册"格式？
   - 是 → 模板 A
   - 否 → 进入问题 3

3. 工程类型是否为基坑/顶管/桥梁迁建（大型工程）？
   - 是 → 仍推荐模板 A
   - 否 → 模板 B 即可

> **保守原则**：拿不准就用模板 A。冗余的章节甲方不会嫌多，缺章节一定挨打。

## 四、立项后启动动作清单

资料初步到位、决策确定后，按顺序做：

- [ ] 项目负责人 / 技术负责人 / 现场负责人三人确定（写入章节 12）
- [ ] 项目编号在公司项目库登记
- [ ] 复用历史相近项目方案：找一份**同类型同等级**的最新方案做基底（**不是给客户的版本**，而是公司内部基线）
- [ ] 创建项目目录（建议结构）：
  ```
  /项目名-年月/
  ├── 01-甲方资料/        ← 原始 PDF、CAD
  ├── 02-方案/            ← Markdown 起草 + 历史版本
  │   ├── v0.1-初稿/
  │   ├── v0.2-内审稿/
  │   ├── v0.3-内审修订/
  │   ├── v0.4-送审稿/
  │   └── v1.0-定稿/
  ├── 03-评审意见/        ← 内审纪要、专审意见原件
  ├── 04-回复表/
  └── 05-附图/
  ```
- [ ] 项目开工会议（kick-off）：与设计 / 施工 / 监理建立联络微信群
- [ ] 用 `references/scoping-and-pricing.md` 做一次快速报价复核（防止合同价不够覆盖工作量）

## 五、常见踩坑

| 坑 | 表现 | 规避 |
|---|---|---|
| **安评报告滞后** | 安评和方案并行编制，安评没出方案先出，结果安评出来后报警值不一致 | 不动笔等安评，或承诺安评出来后整章节重写 |
| **施工筹划版本错** | 拿到的工筹是早期版，开工时已变化，方案阶段划分对不上 | 起草时注明工筹版本号/日期，并在送审前再次确认 |
| **轨道环号弄错** | CAD 里环号标注与现场不一致 | 写信向运营分公司或设计院**书面确认环号起讫**，纪要留存 |
| **运营 vs 建设单位口径不一致** | 同一线路运营段、建设段口径冲突 | 邮件抄送运营分公司+建设分公司双方，让对方确认归口 |
| **甲方反复改工况** | 写到一半甲方说"基坑改成顶管了" | 在合同里写明"重大方案变更需重新计费" |
