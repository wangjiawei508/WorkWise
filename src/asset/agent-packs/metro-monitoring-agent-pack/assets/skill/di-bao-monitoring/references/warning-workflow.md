# 预警、报警与消警工作流

本文件用于编写预警单、报警单、红色预警汇报、数据说明、专题会议材料和消警申请。

## 触发分类

| 级别 | 常见条件 | 常见输出 |
|---|---|---|
| 预警 | 达到预警值，未达到报警值 | 预警单、日报提示、加密关注 |
| 报警 | 达到报警值，或触发速率/数据报警 | 报警单、复测、加密监测、处置建议 |
| 红色预警 | 达到/超过控制值，或出现严重异常 | 红色预警汇报、专题会、专家研判 |
| 消警 | 数据稳定并满足消警条件 | 消警申请、消警依据、附件资料 |

若本项目方案或运营单位对级别名称和阈值有专门定义，以项目要求为准。

## 预警资料必填信息

使用 `assets/warning-notice-template.md`。

| 字段 | 必填内容 |
|---|---|
| 项目 | 当前项目全称 |
| 级别 | 预警、报警、红色预警、消警 |
| 部位 | 车站、区间、线路、桥墩、桥面、施工区域 |
| 监测项目 | 沉降、位移、倾斜、收敛等 |
| 点号 | 所有触发点 |
| 数据 | 本次变量、速率、累计值、阈值 |
| 首次触发时间 | 日期、时间和报告期号 |
| 施工工况 | 触发时段的关键施工内容 |
| 原因分析 | 数据趋势、施工关系、复测核查 |
| 处置措施 | 通知、加密、巡视、复测、施工核查、专家会 |
| 附件 | 曲线、报表页、原始数据、照片、会议纪要、复测记录 |

## 处置措施库

根据级别选择措施：

- 通知建设单位、运营单位、施工单位、监理单位等相关方。
- 加密自动化或人工监测频率。
- 复核仪器、基准点、工作基点、棱镜、静力水准、人工路线。
- 对轨道、隧道、车站、桥墩、桥面和施工面进行现场巡视。
- 对比自动化和人工结果。
- 要求施工方核查开挖深度、堆载、盾构参数、注浆、桩基、降水、支撑拆换等工况。
- 红色预警或超控制值时组织专题会或专家评审。
- 将处置过程写入后续日报、周报和月报。

## 消警逻辑

没有证据时不能写消警结论。消警通常需要：

- 监测值在规定观察期内稳定。
- 无继续不利发展趋势。
- 必要时有人工复核或复测结果。
- 关键风险工序已经完成或风险受控。
- 建设、运营、施工等相关方履行确认流程。
- 附件能证明近期曲线、报表和复测情况。

## 措辞要求

- 用数字说话：点号、数值、阈值、日期必须清楚。
- 区分本次变化、变化速率、累计变化和阈值。
- 证据不足时不要断言施工原因，可用“可能与……有关”“结合施工工况判断……”“建议进一步核查……”。
- 写明事件是首次预警、持续预警、升级、降级还是消警。
