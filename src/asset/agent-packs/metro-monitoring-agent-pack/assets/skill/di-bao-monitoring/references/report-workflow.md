# 报表工作流

本文件用于指导地保监测日报、周报、月报、监护记录的出具。若用户提供了既有模板，应优先保持既有模板结构；本文件用于补齐流程、字段和校核逻辑。

## 目录

- 日报
- 穿越工况高频快报
- 周报
- 月报
- 既有报表修改
- 通用校核

## 日报

### 输入资料

- 本期日期、天气、温度、施工工况。
- 日报编号；若需递推，必须读取上一期报表。
- 当日监测 Excel、PDF、CSV 或系统导出数据。
- 人工监测起止日期；人工未测时不要编造。
- 是否预警/报警及相关处置指令。
- 编写、校核、审核等签字栏信息。

### 常见 Excel 结构

许多日报工作簿具有“辅助表 + 输出表”的结构：

| 表页 | 用途 |
|---|---|
| `info`、`监测日期`、`适配设置页` | 项目元数据、日期、期号、天气、施工工况、报警状态、取数规则 |
| `Z1`、`Z2` | 自动化沉降/高程、水平位移坐标数据 |
| `R1`、`R2` | 人工沉降/高程、水平位移坐标数据 |
| `累计数据（自动化）`、`累计数据（人工）` | 跨期累计数据和曲线数据 |
| `统计表`、`统计分析` | 汇总表、最大值和统计结果 |
| `封面`、`分析报告`、`前台` | 报告可见输出页 |
| 各单项表页 | 单项监测日报表 |

有辅助表时，不要只看封面或可见输出页。可见页可能只是公式结果，且可能残留上一期内容。

### 日报结构

使用 `assets/daily-report-template.md`。常见结构：

1. 封面：项目名、编号、日期、签字栏、监测单位。
2. 监测分析报告：天气、施工工况、监测项目数据统计、正负号备注。
3. 前台/综合分析页：按上行线、下行线、车站、区间、桥面、桥下、桥墩等分区。
4. 单项监测日报表：每个监测项单独成表。
5. 结论与建议：正常、预警、报警分别写不同结论。

### 日报校核

- `本次最大变化量` 和 `累计最大变化量` 必须分别统计。
- 自动化数据可能为 4 小时/次或 6 次/天，应按项目模板确定日报统计口径。
- 人工监测可能不是当天数据，必须注明人工监测实际日期。
- `是否综合报警` 与 `是否数据报警` 不能合并。
- 单项表页的仪器编号、监测间隔、首次/上次/本次日期不得残留旧项目内容。

## 穿越工况高频快报

适用于盾构、顶管、管廊、桥梁或其他风险源穿越轨道交通控制保护区，要求 15 分钟、2 小时、4 小时等高频出报的场景。自动化全站仪/测量机器人按 `references/crossing-total-station-report.md` 处理；上海华桓静力水准沉降按 `references/crossing-static-level-report.md` 处理。

### 输入资料

- 本次出报间隔：15 分钟、2 小时、4 小时或项目指定间隔。
- 初始采集时间、上次监测时间、本次监测时间、报告统计时段。
- 自动化数据源：全站仪平差成果或上海华桓静力水准接口数据、点号、监测项、本次变化量、累计变化量、变化速率。
- 初始值、上次值、本次值或已计算的本次/累计变量。
- 施工工况：盾构/顶管环号、里程、穿越位置、正投影区段、风险工序。
- 点位布置图、影响范围示意图、现场照片；缺照片时保留图位并说明缺口。
- 本项目预警值、报警值、控制值及速率/趋势报警规则。

使用 `scripts/summarize_crossing_total_station.py` 时，默认需要在数据行中提供预警值、报警值、控制值；只有项目明确允许时才使用 `--allow-inferred-thresholds`。

### 报表结构

使用 `assets/crossing-total-station-report-template.md`。参考样例常见为 A3 横版左右两栏：

1. 页眉：项目全称。
2. 一、施工工况：简述工程概况、穿越关系、风险源位置。
3. 二、施工进度：写本时段盾构/顶管/施工推进位置。
4. 图像区：放点位布置图、影响范围图、现场照片或视频截图。
5. 三、各项数据分析：写初始采集时间、上次监测时间、本次监测时间、监测方式。
6. 数据表：按上行线/下行线/左右线/结构分区，列道床沉降、结构沉降、水平位移、水平收敛等。
7. 最大值区：分别列本次最大和累计最大，不能合并。
8. 备注：正负号说明、阈值说明、色标说明。
9. 四、结论与评价：写数据稳定性、报警状态、施工建议和下一次出报频率。

### 色标规则

- 黄色：当前穿越正投影或重点影响区段。
- 绿色：本次或累计绝对值最大、需要关注但未达到预警/报警的数据。
- 红色：达到本项目预警、报警、控制值或触发速率/趋势报警的数据。
- 灰色：可用于非当前重点区段、暂停观测区或模板既有辅助区，但必须在备注中说明。

### 高频报表校核

- 当前快报不得使用晚于本次监测时间的数据。
- 15 分钟报以最新完整观测批次为主；2 小时、4 小时报可增加本时段最大值或趋势描述，但表内时间点要写清。
- 缺测、遮挡、离线、反射片异常不能填 0；使用 `/` 或空值并写明原因。
- 自动化数据若超过一个出报间隔未更新，应在结论中说明“数据未按频率返回”，不能直接写“监测数据平稳”。
- 阈值缺失时不套用旧项目阈值，状态写“待确认阈值”。

## 周报

### 输入资料

- 周报统计期。
- 本周日报或汇总 Excel。
- 本周监测工作量、巡视、会议、联系单。
- 本周施工工况和下周计划。
- 预警、报警、复测、消警事项。
- 现场照片或巡视记录（如模板要求）。

### 周报结构

使用 `assets/weekly-report-template.md`。常见结构：

1. 封面：项目名、周报编号、统计期、监测单位、日期。
2. 本周监测工作量统计：日常监测、巡视巡查、会议、联系单、下周计划。
3. 外部工程/现场施工工况：本周进展和下周风险工序。
4. 监测数据汇总：本期最大变化、累计最大变化、状态。
5. 存在问题及建议：预警报警、数据异常、施工建议。
6. 下周工作计划。

### 周报校核

- 工作量单位必须沿用源表：`4h/次`、`6次/天`、`1次/周`、`点次`、报告份数等不能混用。
- 周报章节名称可能叫“外部基坑现场工况”“施工工况”“现场巡视”，编辑既有报告时尽量保留原称。
- 自动化与人工对比只在同一监测对象、同一监测项、时间区间可比时写。
- 若存在预警/报警，周报“存在问题及建议”必须承接日报或预警单。

## 月报

### 输入资料

- 月报统计期，注意自然月、合同月、实际报表期可能不同。
- 本月日报、周报、汇总表、曲线图。
- 控制网复测报告。
- 监测工作量和项目管理工作记录。
- 预警、报警、消警资料。
- 巡视记录、现场照片、附件资料。

### 月报结构

使用 `assets/monthly-report-template.md`。常见结构：

1. 工程概况及进度。
2. 本月工作开展情况。
3. 控制网复测情况。
4. 监测数据汇总及分析。
5. 监测预、消警情况。
6. 存在问题及建议。
7. 巡视。
8. 附件。

### 月报校核

- 控制网若复测，必须写明高程和平面成果是否沿用或更新。
- 没有控制网报告时，不要写“各工作基点稳定”。
- 曲线图缺失时保留图位并列明所需图表，不要编造曲线结论。
- 附件清单应包含开工时间确认单、监测报表、控制网成果、预警资料等。
- 月报工程概况可以继承上一期，但施工进度、数据、工作量、预消警必须更新。

## 既有报表修改

用户要求“改一下这份周报/月报/日报”时：

1. 保持既有文档结构、标题层级、表格顺序和签字栏。
2. 局部替换项目资料、日期、期号、工况、数据、预警状态。
3. 检查旧项目名、旧单位名、旧日期、旧报告编号是否残留。
4. 若用户要求新版模板，再使用 `assets/` 下的通用模板重写。

## 通用校核

- 项目名、单位名、合同号、日期、期号属于当前项目。
- 自动化和人工统计期分开说明。
- 本次最大、累计最大、报警状态可追溯。
- 正负号说明完整。
- 阈值和状态一致。
- 签字栏和附件清单完整。
