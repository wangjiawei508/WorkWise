# 地保监测报表规则

本 Skill 是 `di-bao-monitoring` 的轻量报表入口。日报、周报、月报、应急快报和消警材料都必须先满足这里的硬规则，再进入具体模板。

## 必做顺序

1. 建立 `project_profile`：项目全称、线路/结构、外部施工类型、报告期、监测单位、施工工况、预警状态。
2. 判断报表类型：日报、周报、月报、应急快报、消警说明。
3. 加载 `di-bao-monitoring/references/report-workflow.md`、`monitoring-catalog.md`、`data-processing.md`。
4. 优先使用 RAILWISE-OS 锁定数据；无锁定数据时再使用用户上传的 Excel/CSV。
5. 输出前检查点号、正负号、阈值、自动化/人工口径和预警状态。

## 日报多子表

日报不得只写一个汇总段。至少检查并按实际项目保留以下子表：

| 子表 | 点号示例 | 说明 |
|---|---|---|
| 自动化结构沉降 | `SDz`、`XDz` | 自动化沉降点，区分上/下行或左/右线 |
| 自动化结构水平位移 | `SJz`、`XJz` | 自动化位移点 |
| 自动化静力水准 | `Swz`、`Xwz` | 静力水准沉降成果 |
| 人工结构沉降 | `SD`、`XD` | 人工水准复核 |
| 人工水平位移 | `Sw`、`Xw` | 人工全站仪/位移复核 |
| 远端/附属监测 | `SYD`、`XYD` | 远端沉降、桥墩、桥面或项目自定义点 |

项目没有某类监测时写“本期无该项监测”，不要虚构表格。

## 周报四件套

周报必须包含：

1. 本周工作量统计。
2. 本周监测数据汇总。
3. 自动化与人工监测对比。
4. 下周计划与风险提示。

自动化和人工监测周期不一致时分别列统计期，不能混成一个最大值。

## 数值口径

- 正负号按本项目方案或报表说明执行。没有说明时标记 `{{待确认正负号约定}}`。
- “本次最大”和“累计最大”通常不是同一个点，必须分别统计。
- 预警优先级：红色报警 > 报警 > 预警 > 正常 > 待确认阈值。
- 阈值来源优先级：项目方案/安评/专家意见/运营单位指令 > 用户给定阈值 > 待确认。
- AI 只写说明和结论，不得改点号、数值、阈值和日期。

## 交付检查

输出前逐项核对：

- 点号和方案/布点图一致。
- 自动化和人工数据来源清楚。
- 每个异常点都有本次值、累计值、速率、阈值和状态。
- 结论能回到数据表或 OS 锁定预览。
- 附件路径、图表、原始表或 JSON 质检报告已列明。
