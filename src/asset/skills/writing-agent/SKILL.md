# writing-agent

这是对 `dongbeixiaohuo/writing-agent` 的 WorkWise Skill 包装。它内置上游写作插件的核心资料：`agents/`、`skills/`、`styles/`、`workflows/` 和 `templates/`。

## 使用场景

在以下需求中使用：

- 用户要写中文长文、观点文、公众号文章、演讲稿或深度分析。
- 用户不想一次性生成全文，而是想分阶段确认方向、素材、风格和成稿。
- 用户要求“去 AI 味”“更像人写”“按某种风格写”。

## 快速模式

收到写作请求后，先判断用户是否已经给出明确模式：

- 若用户明确要“直接写 / 快速出稿 / 短文”，走轻量模式。
- 若用户要长文、深度文章、公众号或需要案例支撑，走协作模式。
- 若用户没有选题，走选题模式。
- 若用户没有明确模式，给出 A/B/C 菜单让用户选，不要直接写全文。

## 工作资料

- `workflows/collab_v2.json`：协作模式的阶段契约。
- `agents/`：每个阶段的角色说明。
- `skills/工作流导演/SKILL.md`：上游主入口规则。
- `skills/风格建模/SKILL.md`：提取或建立文风时使用。
- `skills/公众号文章获取/SKILL.md`：需要提取公众号文章样本时使用。
- `styles/`：上游内置风格样本。

## 执行原则

1. 不要一上来直接生成完整长文，除非用户明确要求快速模式。
2. 协作模式必须先确认：主题、目标读者、文章用途、观点倾向、素材来源、风格要求。
3. 如果可用子代理工具，就按 `workflows/collab_v2.json` 调度；如果没有子代理工具，就在当前会话中模拟这些阶段，并把每一阶段的产物清楚列出。
4. 风格要求必须显式确认。用户说“你来定”时，可以提出 2-3 个候选风格让用户选择。
5. 最终稿之前必须做一次审稿和一次去 AI 味检查。

## 输出约定

轻量模式默认输出：需求复述、短提纲、正文、可选修改建议。

协作模式默认分阶段输出，每阶段结束时给出：阶段名称、阶段产物、下一步选项。
