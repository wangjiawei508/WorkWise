# 全流程 / 阶段门禁 / 异常处理

## 流转强制规则

1. 未收到用户 `确认通过`，禁止开启下一步；
2. 用户 `驳回重做：原因` → 立即退回当前步，按驳回说明重做，不沿用旧版；
3. 用户 `局部修改：XX改为XXX` → 仅改指定段落/单元格，其余已确认内容不变；
4. 九步全部走完且用户终审完毕，项目才闭环，中途禁止新开其他项目。

## 阶段门禁（真实标书不可跳过）

1. **资料完整门禁**：确认招标文件、附件、图纸、澄清通知、模板、响应表格、提交规则齐备；记录缺失/不可读材料。
2. **需求提取门禁**：产出评分/资格/技术/商务/格式/证据台账；识别否决条款、硬性截止、强制表格、签章规则；高风险解读请用户确认后再撰写。
3. **目录/提示词门禁**：MECE 目录，每个评分项与强制项映射到章节/证据/附件；写好每章提示词。目录与提示词被接受前不大规模撰写。
4. **章节撰写门禁**：每章一文件；需图表用 HTML；每章末自查（覆盖需求、假设、证据缺口、后续建议）。
5. **合并/质检门禁**：仅合并已检查章节；跑脚本级检查；人工核最高风险项（评分标准、资格、不可偏离条款、必备表格）。
6. **专家评审门禁**：五专家评审（合规官/技术架构师/评分评委/交付运维/商务法务），有评分办法时按其打分，评论转整改计划。
7. **交付门禁**：按用户与招标要求产出终稿 DOCX/PDF/HTML；准备附件清单、签章清单、未决风险表、文件清单。

## 异常场景

1. **补充/澄清文件**：单独解析补充内容 → 输出「影响范围清单」（哪些已定稿章节/表格受影响及程度）→ 用户决定是否重做受影响章节 → 补充内容并入数据源，同步更新三表、提示词、风控台账；已定稿目录/章节仅用户明确要求时重做。
2. **招标资料缺失**：在对应表格/方案位置标注【招标资料缺失，待补全后完善】，不凭空填充。
3. **跨流程写作请求**：回复"当前处于第X步，需本步确认通过后才可开启后续"。
4. **价格/报价部分**：本 skill 聚焦技术标，报价不在产出范围；如需报价框架由人工另行处理。

## 人工确认点（必须显式确认）
- 目录/提示词映射将决定整体结构时；
- 需在冲突解读间抉择时；
- 某要求看似不可能/高风险/依赖缺失证据时；
- 某论述需证书/授权/案例/价格/人员承诺/法律陈述时；
- 终稿准备转换或打包提交时。
