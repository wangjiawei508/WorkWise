# 撰写提示词库 + 风控台账（步骤④）

两份文件**物理分离**，按目录编号一一绑定。撰写提示词供步骤⑤撰稿（可读）；风控台账仅供步骤⑦质检（撰稿时严禁加载）。

## 一、提示词层级分类

| 层级类型 | 定位 | 提示词侧重 |
| --- | --- | --- |
| 概述型 | 评分项/技术领域总体介绍 | 内容范畴+段落结构+承上启下 |
| 详细设计型 | 技术子领域展开 | 内容范畴+段落结构+图表要求 |
| 逐条响应型 | 对应技术响应表单条(T-NNN) | 技术要求编号+响应方式+响应内容 |

逐条响应型是最低粒度，每条对应技术响应表一行，不可多对一。

## 二、提示词模板

### 概述型/详细设计型
```
【目录编号】3.2
【章节标题】核心层网络详细设计
【章节类型】详细设计型
【内容范畴】①概述核心层设计目标与原则；②引出下文设备选型与配置
【内容边界】仅概述核心层，具体选型详见3.2.1-3.2.3；不涉及汇聚/接入层
【建议字数】300-500字
【段落结构】总-分式：第1段概述目标，第2段说明定位，第3段引出下文
【图表要求】建议配1张核心层拓扑图
【下级章节】3.2.1、3.2.2、3.2.3
【招标原文锚点】技术区-003、技术区-004
```

### 逐条响应型（核心）
```
【目录编号】3.2.1
【章节标题】核心交换机选型与配置
【章节类型】逐条响应型
【技术要求编号】T-001
【招标技术原文】核心交换机：背板带宽≥2Tbps
【招标参数要求】背板带宽≥2Tbps
【响应方式】参数响应
【响应内容】①说明选型依据与方案；②逐项响应背板带宽参数；③描述部署位置与角色
【内容边界】仅响应T-001，其他核心交换机参数详见3.2.2、3.2.3
【建议字数】400-600字
【段落结构】问题-方案式：第1段说明招标要求，第2段选型与参数响应，第3段部署方案
【图表要求】无
```

## 三、响应方式对照

| 技术要求类型 | 响应方式 | 提示词写法 |
| --- | --- | --- |
| 量化参数(≥2Tbps) | 参数响应 | 逐项响应XX参数，说明如何满足 |
| 定性要求(7×24运维) | 方案响应 | 描述运维方案，说明如何满足 |
| 设备/系统选型 | 选型响应 | 说明选型依据、方案、参数对比 |
| 功能要求 | 功能响应 | 描述实现方案与技术路径 |
| 实施类（工期 XX 天） | 实施响应 | 描述实施计划，说明如何按期完成 |

## 四、提示词禁止内容
禁止写入评分分值、扣分规则、废标风险标签和内部评审结论。招标技术参数、实质性要求、验收条件及其“必须/不得”等原文约束必须保留，并通过 T-NNN 与技术响应表一一对应；不能把“必须≥500TB”弱化成“阐述存储容量方案”。

## 五、风控台账模板
```
【目录编号】3.2.1（与提示词一致）
【章节标题】核心交换机选型与配置
【章节类型】逐条响应型
【技术要求编号】T-001
【对应评分条目】P-005：网络架构及设备选型
【招标技术硬性约束】T-001：背板带宽≥2Tbps
【废标风险提示】本章无废标风险
```
硬约束仅提取带 `≥ ≤ 不少于 不超过` 的参数；废标风险重点标 ★号与实质性要求；无则填"本章无……"。

## 六、编号绑定规则
编号一致、逐条对应（目录每编号在两份文件各一条不遗漏）、层级一致、顺序一致、独立存档不可合并。

## 禁止行为
- 风控台账中的内部评分、风险标签和评审笔记混入提示词或正文
- 因为某项要求同时出现在风控台账，就把对应技术参数或验收条件从响应中删除
- 提示词提及评分分值/废标条款
- 套用无项目针对性的通用模板
- 提示词与台账编号不一致、遗漏目录任一编号
- 逐条响应型缺 T 编号、多条要求合并为一条提示词
