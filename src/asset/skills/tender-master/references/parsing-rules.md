# 招标文件解析规则（步骤①细则）

## 一、原文清洗规则

### 1.1 必须剔除（与技术方案无关）

| 剔除类别 | 识别特征 |
|---------|---------|
| 招标公告发布信息 | "招标公告""项目编号""采购人""采购代理机构"公告头 |
| 报名与缴费信息 | "报名费""招标文件售价""投标保证金""开户银行""账户" |
| 开标信息 | "开标时间""开标地点""投标截止" |
| 联系方式 | "联系人""电话""邮箱""地址"且非技术联系人 |
| 无效政策引言 | "根据《XX法》""依据XX规定"且后续无具体执行条款的纯引言 |
| 评标委员会组成 | "评标委员会""评委""评审专家"组成规则 |

### 1.2 必须保留

| 保留类别 | 识别特征 |
|---------|---------|
| 投标人须知 | "投标人须""应符合"的实质性要求 |
| 资格条件 | "资格要求""资质条件""准入条件" |
| 评分办法 | "评分""打分""得分""分值""评审因素" |
| 技术规格 | "技术参数""性能要求""规格""配置""功能要求" |
| 废标条款 | "废标""无效投标""否决""拒绝""不予受理" |
| 标书格式要求 | "装订""页码""盖章""签字""文件组成""格式" |

### 1.3 边界判定

| 情境 | 处理 |
|------|------|
| 一段既有废标又有评分条款 | 两区都保留，各标交叉索引 |
| 既涉技术参数又涉评分条件 | 按主旨归类，另一区加交叉引用"详见X区第X条" |
| 表述模糊难判 | 优先归废标区（宁可多识别风险），标【分类待确认】 |
| "保证金缴纳不足导致废标" | 归废标区 |

## 二、三分法拆分格式

### 技术评分细则区（评分区-NNN）
纳入：技术评审章节全部条款、含明确分值的技术条款、技术类加扣分条款。排除：商务/价格评分。
```
【条目编号】评分区-001
【原文条款】（逐字摘录，不改写）
【分值】X分
【得分条件】（满足什么得满分/部分满足得多少）
【佐证材料要求】（证明文件/检测报告/证书；无则"原文未明确"）
【加/扣分规则】（无则"原文未明确"）
【来源索引】第X章第X节/第X条
```

### 技术需求规范区（技术区-NNN）
纳入：技术参数/性能/功能/配置、服务期/质保/工期/交付、验收标准、软硬件配套。排除：纯商务条款。
```
【条目编号】技术区-001
【原文条款】（逐字摘录）
【参数/要求类别】设备参数/服务标准/工期/运维/交付/软硬件配套
【关键参数】（量化指标：数量、规格、性能数值、时间节点）
【是否带★号或实质性要求】是/否
【来源索引】第X章第X节/第X条
```

### 废标否决条款区（废标区-NNN）
纳入：废标/无效/否决条款、★号实质性要求不满足即废标、资格审查否决项、强制性"必须/应当/须"资格条款（标【需人工确认是否废标】）。排除：商务/价格废标。
```
【条目编号】废标区-001
【原文条款】（逐字摘录）
【废标类型】资格废标/资料缺失废标/格式错误废标/实质性不响应废标
【触发条件】（缺什么/不满足什么/做错什么会废标）
【是否带★号或"实质性要求"】是/否
【来源索引】第X章第X节/第X条
```

## 三、输出结构 `01_requirements/招标文件解析.md`
```
# 招标文件解析
## 一、技术评分细则区
## 二、技术需求规范区
## 三、废标否决条款区
## 四、交叉引用索引
| 条目编号 | 条款简述 | 同时涉及的其他区 | 备注 |
```

## 禁止行为
- 自行删减招标硬性条款、人为合并多条规则（每条独立摘录）
- 保留商务/价格评分条目、改写招标原文
- 凭空推断原文未明确的内容
