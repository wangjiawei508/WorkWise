# 穿越工况自动化全站仪高频报表

本文件用于盾构、顶管、管廊、桥梁等风险源穿越轨道交通控制保护区时，按 15 分钟、2 小时、4 小时或项目指定频率出具自动化全站仪监测快报。当前只处理全站仪/测量机器人数据；静力水准另设流程后再接入。

## 触发场景

- 用户要求“15 分钟一报”“2 小时一报”“4 小时一报”“穿越快报”“自动化全站仪报表”。
- 盾构、顶管或其他施工体进入上穿/下穿/邻近正投影影响区。
- 运营单位或监测方案要求穿越窗口加密出报。
- 已有样例为 A3 横版、左右分栏、上部工况图像、下部数据表的快报格式。

若用户提供的是地下盾构穿越既有地下区间的工程师模板，且报表同时包含全站仪平面变形和静力水准沉降，例如通途西路类模板，应先读 `crossing-underground-shield-combined-report.md`，按联合报表生成；本文件只作为全站仪取数和计算口径参考。

## 必需输入

| 类别 | 内容 |
|---|---|
| 项目资料 | 项目全称、涉及线路、区间/车站/桥梁结构、监测单位、施工单位 |
| 平台取数 | 平台URL、项目ID/prjid、平差网ID、登录方式、数据模块口径 |
| 出报口径 | 出报间隔、报表截止时间、报告编号、平台初始值来源、上次监测时间、本次监测时间、报告统计时段 |
| 施工进度 | 盾构/顶管环号、里程、穿越位置、正投影区段、当前风险工序 |
| 数据资料 | 自动化全站仪平台初始值、当前平差成果、上一平差成果、本次变化量、累计变化量、速率 |
| 监测分区 | 上行线、下行线、左右线、正投影区、延伸范围、非重点区 |
| 阈值资料 | 预警值、报警值、控制值、速率或连续同向报警规则 |
| 点名映射 | 平台原始点号、报表显示点号、桥墩上下点配对、倾斜率点号，经人工确认 |
| 图片资料 | 点位图、影响范围图、现场照片或视频截图 |

缺任何一类资料时保留占位符或在备注列出缺口，不要从参考项目臆造。

当用户只说“做穿越期地保监测报表/快报”，先要求其提供或确认以下信息：

```text
请提供平台URL、项目ID/prjid、平差网ID、登录方式（DIBAO_PLATFORM_USER/DIBAO_PLATFORM_PASSWORD环境变量或浏览器Cookie）、出报频率、报表截止时间、当前施工工况、点名映射确认表、左/右示意图或照片、项目实施单位来源文件。自动化初始值将从全站仪自动化平台初始值接口取X/Y/H，不使用人工初始值报告起算；平台取数只使用报表截止时间前已完成的平差批次。
```

## 出报频率口径

| 间隔 | 报表侧重点 | 数据口径 |
|---|---|---|
| 15 分钟 | 快速判断穿越窗口是否稳定 | 使用本次截止时间前最后一个完整观测批次；重点写缺测、报警、最大值 |
| 2 小时 | 阶段性汇总穿越影响 | 表内仍用本次批次数据，可补充本时段最大值、缺测次数、趋势 |
| 4 小时 | 半班次或班次交接汇报 | 增加工况回顾、照片、趋势判断和下一时段建议 |

若自动化数据返回时间与出报时间不完全一致，以“本次监测时间”写明实际数据批次；不得用晚于出报截止的数据补当前报表。补跑历史报表时必须显式传入报表截止时间，例如补 2026-06-04 16:00 报表，应锁定 `2026-06-04 16:00:00` 之前最后一个已完成批次，而不是使用之后才完成的批次。

## 数据整理

1. 平台取数优先走 `数据查询 / 平差与测站数据 / 变形监测平差数据`：先确认报表截止时间之前最新已完成平差批次，再取该批次对应的平面/高程平差报告和点位累计变形。不要直接把首页“最新动态”、告警详情、晚于出报时刻的平差批次或未锁定批次的当前点位表当成最终来源。
2. 保留通用字段：`monitoring_item`、`monitoring_method`、`structure_zone`、`point_id`、`current_change_mm`、`rate_mm_per_d`、`cumulative_mm`、阈值列。
3. 增加高频字段：`report_cadence`、`report_start`、`report_end`、`initial_time`、`previous_time`、`current_time`、`ring_no`、`position_label`、`influence_zone`、`validity`。
4. 先判别点位角色：`CZ` 前缀为测站，`JD` 前缀为测量基点/基准点，不作为监测点，不参与监测点数量、本次最大、累计最大和数据表统计。
5. 先建立点名映射表：当平台原始点号与工程师快报点号不一致时，按编号、布点图和初始值报告建立映射，例如 `S12/X12 -> MCC012`、`S13/X13 -> MCC013`、`S12/X12 -> MCQX012`。映射表必须经用户/工程师人工确认后才可用于正式报表；未确认时只输出核对草稿。模板见 `assets/point-alias-map-template.json`。
6. 自动化全站仪快报必须从全站仪自动化平台获取平台初始值。当前平台可从 `ConfigManage/MonitorConfig/Point/PointHandler3.ashx?title=get&pointtype=1` 读取监测点配置中的 `X/Y/H`，该组值作为累计变形的起算坐标；不得用人工初始值报告替代。人工初始值报告只用于提取项目实施单位等文字信息。
7. 如果需要自动抓取平台平差后成果，应优先用环境变量或 Cookie 登录。不要把账号密码写进报告、CSV、PDF 或最终回复。命令示例：

```bash
export DIBAO_PLATFORM_USER="{{用户名}}"
export DIBAO_PLATFORM_PASSWORD="{{密码}}"

python scripts/fetch_adjusted_total_station.py \
  --base-url "{{平台URL}}" \
  --prjid "{{项目ID}}" \
  --adjust-net-id "{{平差网ID}}" \
  --report-cadence "{{15min/2h/4h}}" \
  --report-cutoff-time "{{报表截止时间}}" \
  --data-grace-minutes "{{整点后取数宽限分钟，常用5}}" \
  --previous-report-cutoff-time "{{上期报表截止时间}}" \
  --previous-data-grace-minutes "{{上期整点后取数宽限分钟，常用5}}" \
  --cumulative-source "platform-deformation" \
  --current-change-source "previous-batch" \
  --initial-time "全站仪自动化平台初始值" \
  --work-condition "{{当前施工工况}}" \
  --output-dir "平台数据输出"
```

该脚本会抓取 `StationDataHandler.ashx?title=getPage&fpage=AdjustData1` 中报表截止时间前最新已完成平差批次、`AdjustReportChild.aspx?adjsummid={{平差ID}}&rpt=1/2` 的平面/高程报告、`AllPointCurrInfoHandler.ashx?title=get1` 的平台平差变形成果 `dX/dY/dH`、`PointHandler3.ashx?title=get&pointtype=1` 的平台初始值，以及当前/上一期报表对应批次的 `PointMonCoordHandler.ashx?title=get1&showData0=true` 坐标记录，并转换为报表 CSV。默认 `--cumulative-source platform-deformation`，即累计量采用平台平差成果表直接返回的 `dX/dY/dH`；平台初始值 `X/Y/H` 用于高差、点位配置和校核。若平台整点后几分钟才生成当次平差成果，保留名义 `--report-cutoff-time`，另设 `--data-grace-minutes 5` 等宽限值取整点后已完成批次，并在摘要中记录宽限。若计算“日变量/单次变化”，必须传 `--previous-report-cutoff-time` 和 `--previous-data-grace-minutes`，使单次变化按“本期名义窗口成果 - 上期名义窗口成果”计算。沉降取平台垂直位移/Z值 `dH`；水平位移分东西、南北两个方向，东西方向取Y坐标差，Y变大为向东方向位移，南北方向取X坐标差，X变大为向北方向位移；桥墩倾斜也分东西、南北两个方向，东西向倾斜按上下点Y坐标位移差除以上下点初始三维距离，南北向倾斜按上下点X坐标位移差除以上下点初始三维距离。若 Mac 或非 Windows 浏览器页面不显示平差报告，应先用接口响应判断后台是否有数据；后台有平差 ID 和报告正文时，可以继续出报。

8. 有结构化 CSV 时运行：

```bash
python scripts/summarize_crossing_total_station.py input.csv
```

脚本会输出逐行报警判定 CSV 和 Markdown 汇总。正式报表按用户提供的截图样式输出 A3 横向 Excel 时，继续运行；若抓数脚本生成了同前缀 `*_platform_initial_values.json`，报表脚本会自动用它计算上下点高差：

```bash
python scripts/build_crossing_total_station_xlsx.py \
  "平台数据输出/*_adjusted_total_station.csv" \
  --template "{{快报模板.xlsx}}" \
  --initial-report "{{初始值报告.docx}}" \
  --image "{{左图}}" \
  --image-right "{{右图}}" \
  --tilt-axis "纵向变形量" \
  --point-alias-map "{{点名映射.json}}" \
  --manual-overrides "平台数据输出/manual_metric_overrides.json" \
  --work-condition "{{当前施工工况}}" \
  --output "{{输出报表.xlsx}}"
```

若平台初始值 JSON 不在同前缀位置，应显式传入：

```bash
python scripts/build_crossing_total_station_xlsx.py \
  "平台数据输出/*_adjusted_total_station.csv" \
  --point-config "{{平台初始值.json}}" \
  --point-alias-map "{{点名映射.json}}" \
  --image "{{左图}}" \
  --image-right "{{右图}}" \
  --output "{{输出报表.xlsx}}"
```

Excel 快报应采用 A3 横向通栏排版，保留密集黑色表格线、上部双图片区、中部数据分析表、底部最大值/备注/结论区。“施工工况”“施工进度”“各项数据分析”“结论与评价”应横向合并成一栏，不要左右重复分栏；“工况及影响范围示意图”下方左、右各放一张图，图片按图片区裁切填充，不留白、不越界；“平台最新动态”放到图片区下方，做左右两组 4 列信息表。

地下盾构穿越既有地下区间的通途西路类模板不是桥墩倾斜模板。此类报表常见工作表为 `前台`、`前台 (静力水准)`、`Z1 全 (2)`、`Z1 （静力水准)`、`盾构总累计曲线图` 等，监测项为道床沉降、结构沉降、水平位移、水平收敛和静力水准结构沉降。不要生成 `MCC/MCW/MCQX` 桥墩明细页，除非用户明确该项目是高架桥墩。

快报页沉降、水平位移按 0.1mm 显示，桥墩倾斜按 0.01‰ 显示；完整报表的桥墩倾斜明细页坐标和倾斜计算值保留 5 位小数，并生成桥墩沉降、桥墩水平位移、桥墩倾斜三张曲线图。源 CSV 保留原始计算值用于追溯。若工程师对个别点位/监测项做人工复核修正，不要直接改 Excel 成果；应写入 `manual_metric_overrides.json` 一类的覆盖表，至少包含报表时间、平台点号、报表点号、监测项、方向、本次变化值、累计变化值和原因，再通过 `--manual-overrides` 生成，使平台动态、主表、明细公式、最大值、曲线图和留痕一致。

输出文件名必须使用业务命名并带名义报表日期，不要使用技术型前缀、接口批次时间戳或内部口径词。推荐格式：`{{项目全称}}_{{YYYYMMDD}}_{{名义出报小时}}点_4小时快报.xlsx`，例如 `绕城高速管廊工程1标段施工期间轨道交通3号线控制保护区监测项目_20260605_00点_4小时快报.xlsx`；PDF 同名，仅扩展名改为 `.pdf`。其中日期和小时均按报表名义截止时间取值，不按实际触发时间或平台批次时间取值。

默认情况下，脚本要求每行提供 `warning_threshold_mm`、`alarm_threshold_mm`、`control_threshold_mm`。若经项目确认可沿用通用默认阈值，再加 `--allow-inferred-thresholds`。

## 计算规则

- `本次变化量/日变量`：本期名义报表窗口对应的平台成果减上期名义报表窗口对应的平台成果。上期不一定是平台上一条平差批次，应按 `--previous-report-cutoff-time` 锁定上期报表窗口。
- `累计变化量`：当前项目按工程师表坐标公式计算，沉降为 `本次Z-初值Z`，水平位移为 `本次Y-初值Y`；平台累计 `dX/dY/dH` 用于反推并校核自动化初值。单次变化为 `本次平差坐标-上期平差坐标`，补跑时要锁定上期实际平差批次。
- `本次最大`：同一分区、同一监测项内，本次变化量绝对值最大的点。
- `累计最大`：同一分区、同一监测项内，累计变化量绝对值最大的点。
- `变化速率`：优先使用源表给出的速率；无速率时按本项目频率和时间差计算，单位必须说明。
- 水平位移必须沿用本项目正方向；如果源表只有坐标差，先按项目方向轴转换再填报。
- 水平收敛必须明确是成对点差值或系统已计算成果，不能用单点水平位移代替。

## 表格字段

参考样例下部数据表常用结构：

| 栏目 | 说明 |
|---|---|
| 桩号及位置/位置 | 环号、里程、结构区段或点位位置 |
| 监测点号 | 点号，如 S763、X765 |
| 道床沉降 | 本次变量、总累计值、备注 |
| 结构沉降 | 本次变量、总累计值、备注 |
| 水平位移 | 本次变量、总累计值、备注 |
| 水平收敛 | 本次变量、总累计值、备注 |
| 桥墩倾斜率 | 本次倾斜率、累计倾斜率、上下测点初始三维距离或配对说明 |

按项目实际监测项增删列。没有的监测项不要留空列；新增监测项可按“本次变量、总累计值、备注”三列扩展。

高架桥墩倾斜率优先使用平台或人工成果中已经给出的 `MCQX/倾斜率` 字段。若平台仅给出 S/X 上下点平差坐标，则按同一桥墩上下测点成对计算，例如 S12/X12、S13/X13。当前项目按工程师公式：单次倾斜 `((本次S点X-上次S点X)-(本次X点X-上次X点X))/上下点初始三维距离×1000`；累计倾斜 `((本次S点X-初始S点X)-(本次X点X-初始X点X))/上下点初始三维距离×1000`。位移单位为 m，结果数值单位为 ‰；正值为北方向偏移，负值为南方向偏移。上下点初始三维距离用平台自动化初值 `X/Y/H` 计算；缺初值时写“初始三维距离待确认”，不得硬算。

点名映射确认后，报表显示点号使用映射点号，源数据追溯仍保留平台原始点号。当前项目报表可见点名不得出现 `S12/X12/S13/X13` 等平台原始点名：沉降显示为 `MCC012/MCC013`，水平位移显示为 `MCW012/MCW013`，且按下部 `X` 点分别填列东西方向和南北方向；倾斜率配对显示为 `MCQX012/MCQX013`，按对应 `S/X` 上下点配对分别计算东西方向和南北方向。

## 版式骨架

使用 `assets/crossing-total-station-report-template.md` 起稿。参考样例通常按以下顺序：

1. 页眉：项目全称。
2. 一、施工工况：项目关系、线路结构、穿越风险描述。
3. 二、施工进度：本时段施工推进，写清环号、里程或区域。
4. 图像区：通栏标题下左、右各放一张图，可用全图 + 局部放大图，或点位/影响范围图 + 现场照片；章节标题不重复分栏。
5. 平台最新动态：放在图片区下方，左、右各 4 列信息表，包含项目名称、项目实施单位、初始采集、本次/上次、监测方式、平台预警、本次最大、累计最大、倾斜率和说明。
6. 三、各项数据分析：初值采集时间、上次监测时间、本次监测时间、监测方式。
7. 数据表：按上行线/下行线、左线/右线、桥面/桥下等分区。
8. 最大值行：分别列本次最大和累计最大。
9. 备注：正负号、阈值、速率规则、色标。
10. 四、结论与评价：数据稳定性、报警状态、施工建议；底部 footer 写项目实施单位和 `轨道交通控制保护区监测快报`，不保留空边框行。

跨平台导出要求：

- 正式 PDF 尽量固定在同一导出环境生成，例如同一版本 LibreOffice headless 或同一台 Windows Excel。Mac Excel、Windows Excel、WPS、LibreOffice 的字体度量、默认打印机、图片锚点和分页可能不同。
- 模板必须固定 A3 横向、打印区域、缩放到 1 页宽/1 页高、行高列宽和字体；Windows 缺少 `SimSun/宋体` 时应先安装或按项目统一字体替换。
- 交付前以最终 PDF 核对分页、图片裁切、图表位置、表格边框和页数，不以 Excel 编辑界面作为唯一判断。
- 发送给 Windows 用户或打包 zip 前运行 `scripts/clean_report_artifacts.py "{{输出目录}}" --check-archives`，清理 `.DS_Store`、`__MACOSX/`、`._*`、`Thumbs.db`、`desktop.ini`、`~$*` 等缓存或临时文件。

## 色标与备注

- 黄色：当前穿越正投影区或重点影响区段。
- 绿色：只标记主数据区本次/累计绝对值最大但未报警的数据；“本次最大/累计最大”汇总行只写点号和值，不填绿色。
- 红色：达到预警、报警、控制值或触发速率/趋势报警。
- 灰色：非当前重点区、暂停监测区或模板辅助区；必须能解释。

常用备注：

1. 水平位移：“+”为靠近风险源方向，“-”为远离风险源方向；若项目另有定义，以项目说明为准。
2. 竖向位移：“+”表示隆起，“-”表示下沉。
3. 收敛：“+”表示扩张，“-”表示收缩；若模板定义相反，按模板。
4. 预警值、报警值、控制值按本项目方案、安评或运营单位要求填写。

## 结论写法

正常时：

```text
本时段自动化全站仪监测数据总体平稳，各监测项本次变化量和累计变化量均未达到预警值。后续按 {{15分钟/2小时/4小时}} 频率持续监测，并结合施工推进情况及时复核异常数据。
```

存在异常但未报警时：

```text
{{点号}} 本次/累计变化量相对较大，尚未达到预警值。建议继续关注该点及相邻测点趋势，检查现场反射片、视线通视和施工扰动情况。
```

达到预警或报警时：

```text
{{点号}} {{监测项}} {{本次/累计/速率}} 达到 {{预警/报警/控制}} 条件，建议立即通知相关单位，开展数据复核、现场巡视和施工参数核查，并按项目要求加密监测或组织专题研判。
```

## 交付前检查

- 项目名、线路、区间、报告编号、时间均为当前项目。
- 初值、上次、本次时间完整且前后顺序正确。
- 表内点号与点位图、影响区段一致。
- 黄色/绿色/红色标识和备注一致。
- 本次最大与累计最大分别统计，不能合并。
- 阈值来源明确；缺阈值时写“待确认阈值”。
- 缺测、遮挡、离线、异常剔除均有备注。
- 结论承接数据和施工工况，不写无依据的“良好”。
