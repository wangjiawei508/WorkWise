# 初始值采集与项目总结工作流

本文件用于补充地保监测两个容易遗漏但非常关键的阶段：开测前初始值采集/审批、项目完成后的监测总结。

## 一、初始值采集阶段

初始值是后续所有“本次变量、累计变量、报警判定”的基准。没有经过审核确认的初始值，不应直接进入正式日报、周报、月报。

### 1. 前置条件

开始初始值采集前应确认：

- 监测方案已通过内部审查或外部评审，监测项目、点位、频率、阈值已经明确。
- 测点已按方案完成埋设，并形成测点埋设记录。
- 自动化测点、人工测点、控制网基准点/工作基点编号与布点图一致。
- 使用仪器已检定且在有效期内。
- 作业人员、轨行区/高架登高/运营窗口等安全条件满足要求。
- 平面控制网和高程控制网已完成第 1 期联测或已有可沿用成果。

### 2. 常见成果组成

| 成果 | 作用 |
|---|---|
| 测点埋设记录 | 记录测点位置、编号、埋设时间、材料规格、照片、示意图 |
| 测点验收记录 | 由监测、监护、运营/地保等相关方确认测点数量、编号和状态 |
| 监测初始值审批表 | 将各监测项初始高程、坐标、收敛值、倾斜值等提交审核 |
| 平面控制网联测报告第 1 期 | 建立平面坐标系统、首级控制点和工作基点成果 |
| 高程控制网联测报告第 1 期 | 建立高程基准、工作基点高程成果 |
| 点位布设图/照片 | 证明点位与方案一致，便于后续查找和复测 |

### 3. 测点埋设记录要点

参考样例中，测点埋设记录通常包括：

- 项目名称、监测单位、编号。
- 监测项目，如上行线道床沉降、下行线结构沉降、上行线水平位移、桥墩倾斜等。
- 测点数量、测点编号范围、对应监测对象。
- 点号、埋设时间、环号/桥墩号/位置。
- 埋设说明：测钉规格、小棱镜、反射片、倾斜点、埋入深度、固定方式。
- 点位说明或示意图。
- 现场照片。
- 验收人员签字。

### 4. 监测初始值审批表要点

初始值审批表一般按“人工监测初始值成果”和“自动化监测初始值成果”分章。

常见章节：

1. 人工监测初始值成果
   - 上行线道床沉降。
   - 下行线道床沉降。
   - 上行线结构沉降。
   - 下行线结构沉降。
   - 远端沉降。
   - 竖向收敛、水平收敛。
   - 上行线/下行线水平位移。
2. 自动化监测初始值成果
   - 道床沉降。
   - 结构沉降。
   - 水平位移。
   - 收敛或其他自动化项目。
3. 高架/桥梁项目可替换为：
   - 桥墩沉降。
   - 桥墩水平位移。
   - 桥墩倾斜。
   - 桥面/梁端/盖梁/承台沉降。

初始值成果表至少包含：

| 字段 | 说明 |
|---|---|
| 序号 | 连续编号 |
| 测点编号 | 与布点图、日报模板一致 |
| 初始高程/坐标/收敛值/倾斜值 | 按监测项选择 |
| 采集时间 | 初始值日期 |
| 备注 | 异常、未采、复测说明 |

### 5. 平面控制网联测报告要点

平面控制网报告常见结构：

1. 控制网的建立。
2. 测量实施说明。
3. 点位编号说明。
4. 相关技术要求。
5. 测量成果。
6. 成果更新原则。
7. 结论。
8. 附件：原始记录、平差报告、示意图。

重点写清：

- 坐标系统和网形，如独立坐标系、附合导线等。
- 首级基准点和工作基点编号。
- 观测仪器和精度。
- 测角、测距、边长、闭合差等技术指标。
- 成果是否作为后续监测起算数据。
- 后续复测和成果更新规则。

### 6. 高程控制网联测报告要点

高程控制网报告常见结构：

1. 测量方法。
2. 基准点设置。
3. 控制网路线。
4. 控制指标。
5. 控制网测量成果。
6. 成果更新原则。
7. 结论。
8. 附件：水准测量原始记录、平差报告。

重点写清：

- 起算点、基准点、工作基点。
- 水准路线和往返测方式。
- 观测顺序、前后视距差、闭合差要求。
- 平差高程、使用高程、较差、中误差、是否更新。
- 第 1 期初始联测是否沿用原成果或形成新使用成果。

### 7. 初始值阶段校核

交付初始值资料前检查：

- 测点编号、点数、监测项目与方案和布点图一致。
- 人工和自动化初始值分开列明。
- 沉降、位移、收敛、倾斜的单位和正负号定义明确。
- 控制网基准点和工作基点关系清楚。
- 初始值日期早于正式监测日报日期。
- 无缺点、漏点、重复点；若有未采点，写明原因和补采计划。
- 审批表、验收表、原始记录、照片和附件齐全。

## 二、项目总结阶段

项目总结不是月报的简单合订本，而是对整个监测监护周期的完整闭合。应能说明“监测做了什么、数据如何变化、是否保障结构安全、是否满足停测/结束条件、后续有什么建议”。

### 1. 启动条件

一般在以下场景编写总结：

- 外部施工影响阶段完成。
- 项目进入停测申请或已停测。
- 合同监测服务周期结束。
- 运营/建设单位要求提交总结报告。
- 预警事件完成闭合，需要形成阶段性或最终总结。

### 2. 必备资料

- 监测方案和评审意见。
- 起止日期、停测申请或停测确认。
- 日报、周报、月报、监护记录。
- 初始值报告和控制网复测报告。
- 预警、报警、消警资料。
- 巡视记录、现状普查、病害调查、照片。
- 人员和仪器投入记录。
- 施工节点和实际工况。
- 各监测项曲线图和汇总表。

### 3. 总结报告常见目录

参考样例，项目总结可按下列结构组织：

1. 目的及编制依据
   - 目的。
   - 编制依据。
   - 工程概述。
   - 位置关系、周边环境、施工节点。
   - 投入人员及设备。
2. 监测监护总结
   - 监测监护范围。
   - 监测周期及停测。
   - 监测监护工作量统计。
   - 监测内容、精度、频率。
   - 控制网建立与复测。
   - 平面坐标系统和高程基准。
   - 平面控制网、高程控制网。
   - 监测信息反馈。
   - 监测成果分析与评述。
   - 分项节点变形分析。
   - 监护项目及内容。
   - 监护巡查。
   - 各阶段监护实施情况。
   - 安全生产总结。
3. 结论与建议
   - 结论。
   - 建议。
   - 附件：监测项目平面图、曲线、报表清单等。

### 4. 成果分析写法

成果分析应按“总体 + 分项 + 阶段”展开：

- 总体变形情况：各监测项最大累计值、最大速率、是否超预警/报警/控制值。
- 分项分析：结构沉降、道床沉降、水平位移、收敛、桥墩倾斜、桥面沉降等。
- 阶段分析：围护、开挖、支撑、盾构穿越、顶管、桩基、回填、停测前稳定期。
- 预警分析：列明发生时间、点号、数值、原因、处置和消警情况。
- 监护巡查：交底、现状普查、日常巡查、是否发现新增病害。

### 5. 结论写法

结论应回答：

- 监测期间轨道交通结构变形是否处于可控范围。
- 是否发生预警、报警、红色预警；若发生，是否已闭合。
- 控制网成果是否稳定，是否满足监测精度要求。
- 施工各阶段是否按方案开展监测监护。
- 是否满足停测或结束监测条件。
- 后续是否需要继续观察、保留测点、移交资料或注意长期影响。

### 6. 项目总结校核

交付总结前检查：

- 起止日期、施工节点和监测周期一致。
- 工作量统计与日报/周报/月报数量一致。
- 人员设备表与实际投入一致。
- 初始值、控制网、日报、月报、预警资料能互相对应。
- 最大值、曲线、结论数据一致。
- 预警/报警/消警事件闭合。
- 停测依据或结束条件明确。
- 附件完整，图表编号连续。
