# 2 期：内审准备与会议组织

初稿写完不能直接送外审。**先过内审**——这是公司技术质量把关的最后一道防线，也是把外部评审意见"打提前量"的关键环节。

## 一、内审 vs 专审：是什么关系？

| 维度 | 内审 | 专家评审 |
|---|---|---|
| 组织方 | 监测单位（你方公司）技术部门 | 业主 / 运营分公司 / 第三方专家组 |
| 参会人 | 项目组 + 部门技术负责人 + 公司总工 | 业主代表 + 5 位以上外部专家 + 设计 / 施工 / 监理代表 |
| 目的 | 把技术错漏、低级失误挡在出门前 | 决定方案能否通过、能否进场 |
| 输出 | 内审意见单 + 内审纪要 | 评审意见单 + 评审决议 |
| 失败成本 | 重写一次 | 重写 + 进度延误 + 公司信誉受损 |

> **铁律**：**内审必须充分**，能把 80% 的问题在内审挡住，专审就轻松。内审走形式 → 专审被反复打回。

## 二、内审准备（送审前 3 天启动）

### 2.1 自查清单（必做）

按顺序做：

1. **加载 `references/review-checklist.md`** —— 用第三章评审会议高频意见自检表逐条对照
2. **加载 `references/phase1-drafting.md` 第 4 节** —— 用 10 条初稿完工自查
3. **运行下面的"内审预演 20 问"** —— 这是关键

### 2.2 内审预演 20 问

把方案打印或全屏打开，自问下列问题，**每个问题都要能从方案里找到对应章节作答**。答不出的，立即回去补：

**A. 资料覆盖类**
1. 工程总平面图能不能让外人一眼看出新老结构、轨道车站、区间、附属、顶管通道的位置关系？
2. 工程类型描述里有没有给出"工程对轨道交通的扰动机理"？
3. 施工筹划版本号 / 日期是否注明？
4. 安评报告的关键结论（风险等级、最大变形预测值）有没有引用？
5. 既有线 1.7 工后沉降数据有没有具体数字（不是"已稳定"三个字）？

**B. 监测参数类**
6. 监测范围公式（投影 + 两侧延伸值）是否明确？延伸值的依据是什么？
7. 重点监测区在哪？为什么是这里？
8. 监测阶段几个？阶段边界对应的工序节点是什么？为什么选这些节点？
9. 自动化监测频率 6 次/天还是 12 次/天？为什么？
10. 报警值是否分预警 / 报警 / 控制三级？是否分施工阶段细化？

**C. 重难点类（最敏感）**
11. 监测重难点几条？把"本项目"换成"段塘大桥"是否仍然成立？（成立的话——重写！）
12. 每条重难点有没有对应的应对措施？措施是否可执行（含频率、数量、责任分工）？

**D. 控制网与测点类**
13. 控制网有没有平面控制网示意图？
14. 高程工作基点的误差消除方法是什么？
15. 各类型监测点（道床沉降、结构沉降、水平位移、收敛、静力水准）的埋设方法都写了吗？
16. 静力水准基准点位置是否明确？

**E. 应急与配合类**
17. 应急组织机构是否包含地保办—地保车间—监测单位—施工方四级？
18. 三色预警（黄 / 橙 / 红）响应流程图是否齐全？
19. 24h 应急联系电话表是否填完整？

**F. 形式类**
20. 全文是否搜过"待补"、"{"、"XX"等占位符？（最易踩雷）

### 2.3 内审材料组卷

送内审至少要这几样：

- [ ] **方案正文 v0.2**（去除所有占位符）
- [ ] **附图电子稿**（至少有总平图、监测范围示意图、典型断面图三张）
- [ ] **关键资料溯源页**（一页 PPT 或 Word，列出"本方案各关键数据来自哪份资料"）
- [ ] **20 问自查记录**（内审专家看了会很欣赏，相当于自己先质问了一轮）

提交方式：

- 邮件：抄送项目负责人、技术负责人、部门总工，正文写明"申请于 X 月 X 日内审，烦请提前阅读"
- 微信：同步发部门技术群，注明截止时间

## 三、内审会议组织

### 3.1 议程模板

```
{工程全名}监测方案 内审会议

时间：{年-月-日 XX:XX–XX:XX}
地点：{XX 会议室}
主持：{部门技术负责人}

议程：
  1. 项目背景与工程概况说明（项目负责人，10 min）
  2. 监测方案要点汇报（编制人，20 min）
     - 监测范围与等级
     - 监测阶段与频率
     - 重难点与应对措施
     - 报警值
  3. 内审讨论（自由发言，40 min）
  4. 改进意见汇总与下一步计划（主持人，10 min）

参会：
  - 项目负责人：{姓名}
  - 技术负责人：{姓名}
  - 现场负责人：{姓名}
  - 部门技术总工：{姓名}
  - 公司总工（视项目级别决定是否参加）
  - 同部门相关项目经理：{姓名}（横向比对作用）

需提前阅读资料：
  - 方案 v0.2
  - 附图电子稿
  - 20 问自查记录
```

### 3.2 汇报 PPT 结构（20 min）

| 页 | 内容 | 时长 |
|---|---|---|
| 1 | 封面 | — |
| 2 | 工程概况（含一张总平图） | 2 min |
| 3 | 工程与轨道交通位置关系（剖面图） | 1 min |
| 4 | 安评报告关键结论摘要 | 2 min |
| 5 | 监测范围与重点监测区 | 2 min |
| 6 | 监测阶段划分（与工筹对照表） | 2 min |
| 7 | 监测项目与频率（表格） | 2 min |
| 8 | 监测点布设方案（断面图） | 2 min |
| 9 | 报警值表（分阶段） | 2 min |
| 10 | 重难点与应对措施 | 3 min |
| 11 | 工作量与人员设备 | 1 min |
| 12 | 待确认事项与风险点 | 1 min |

**Tip**：第 12 页非常关键——主动把自己心里没底的点列出来让内审讨论，比让别人挑出来强。

### 3.3 内审记录工具

会议期间安排一个人专门**记意见**（不是编制人，让编制人专心答辩）。记录格式：

| 序号 | 发言人 | 意见内容 | 是否采纳 | 落实位置 |
|---|---|---|---|---|
| 1 | {总工} | 监测范围应再扩展 5 环 | 是 | 章节 3.1 |
| 2 | {同部门 PM} | 重难点第 3 条与 2 号线 XX 项目雷同 | 是 | 章节 2.3 重写 |
| 3 | ……  | …… |  |  |

## 四、内审纪要回填

会后 1 个工作日内完成纪要并发回部门会签。纪要模板见 `references/review-response-template.md` 第一节（内审版本）。

要点：

- 每条意见必须有**回复**（"已修改 / 计划修改 / 不采纳，原因 X"）
- 每条采纳意见必须**落到方案的某一页 / 某一节**
- 不采纳的意见要有**充分理由**——通常以引用规范条款或安评结论为据

**内审纪要要双签字**：编制人 + 部门技术负责人。

## 五、内审到送审的过渡

内审通过后，方案变为 `v0.3-内审修订` 版本：

1. 把内审意见逐条改完
2. 全文再通读一遍（找前后不一致——常见于多人协作时）
3. 补齐所有附图（这一步在内审前可以是占位）
4. 签章页填齐（编制 / 校核 / 审核 / 批准）
5. 组卷送审：方案正本 + 附图册 + 内审意见回复 + 内审签字纪要 + 安评报告（如运营方要求附）

**送审材料寄出后再做的事**：

- 把 v0.3 设为分支保留，**任何后续改动开新版本 v0.4**
- 在公司项目库登记送审日期，约定外审时间
