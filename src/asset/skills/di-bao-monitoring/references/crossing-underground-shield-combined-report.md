# 地下盾构穿越既有地下区间联合快报

本文件用于“地下盾构/综合管廊穿越既有地下轨道区间”的联合报表场景。典型特征：

- 全站仪负责平面变形：纵向变形量、横向位移、水平收敛。
- 静力水准负责沉降：结构沉降/道床沉降的竖向变化。
- 报表需同时输出前台快报、静力水准前台、分项数据页和曲线图。
- 适用“通途西路类”地下盾构穿越既有地下区间工程师模板。

## 1. 模板优先级

用户提供的工程师 Excel 模板优先于通用快报模板。遇到此类项目时：

1. 先按工程师模板的 sheet 名、标题、打印区域、行高列宽、合并单元格和图表位置出报。
2. 再按全站仪和静力水准两条数据链分别填充。
3. 不要把它误判成高架桥墩模板，也不要只输出单页全站仪快报。

## 2. 工作簿结构

工程师模板通常包含这些工作表：

| 工作表 | 用途 |
|---|---|
| `前台` | 联合快报首页，左右双线展示上行线/下行线 |
| `前台 (正穿)` | 正穿工况的前台页，结构类似 `前台` |
| `前台 (静力水准)` | 静力水准前台页，展示结构沉降本期/累计、分区表和结论 |
| `Z1 全 (2)` | 全站仪/盾构总累计辅助数据页，含多时点序列 |
| `Z1 （正穿）` | 正穿工况的全站仪辅助数据页 |
| `Z1 （静力水准)` | 静力水准辅助数据页 |
| `盾构总累计曲线图` | 全站仪/盾构总累计曲线辅助页 |
| `日变量曲线图` | 日变量曲线辅助页，通常隐藏 |
| `注浆15分钟` | 其他专题页，通常隐藏或仅在特定项目启用 |

不要把用户提供的工程师原始 workbook 直接提交到公开仓库。公开 skill 中只沉淀模板结构、字段和计算口径；项目私有模板作为运行时输入或本地资产管理。

## 3. 前台页版式

### 3.1 通用要求

- `前台` 和 `前台 (正穿)` 为左右双列布局。
- 左侧写上行线，右侧写下行线。
- 通途西路工程师样表中 `前台` 常见打印区域为宽幅双列区域，`前台 (静力水准)` 常见为单页沉降区域；实际生成时以用户最新模板的打印区域为准，不强行套桥墩模板的 `全站仪快报` sheet。
- `工况及影响范围示意图` 需做成通栏图片区，图片按区域裁切填充。
- `平台最新动态` 放在图片区下方，左右各 4 列信息，不要再拆成左右两个独立表头段。
- `施工工况`、`施工进度`、`各项数据分析`、`结论与评价` 只保留一套内容，不要做成两列重复内容。

### 3.2 `前台` 关键内容

常见段落：

- 工程概况：项目全称 + 当前施工情况。
- 施工进度：当前环号、推进完成、开始拼装等。
- 各项数据分析：上次监测时间、本次监测时间、监测项目表。
- 结论与评价：监测总体稳定、是否预警、后续频率。

通途西路样表中可见字段示例：

- 上次监测时间、本次监测时间。
- 桩号及位置、环号、监测断面、监测点号。
- 单次变化量、穿越期间累计值、总累计值、备注。
- 本次最大、穿越期间最大、总累计最大。

监测项常见为：

- 道床沉降
- 结构沉降
- 水平位移
- 水平收敛

### 3.3 `前台 (静力水准)` 关键内容

静力水准页只写沉降相关内容：

- 结构沉降本次变化量
- 穿越期间累计值
- 本次最大/穿越期间最大
- 结论与评价

不要混入水平位移或倾斜列。

## 4. 数据页口径

### 4.1 `Z1 全 (2)`

该页用于全站仪/盾构总累计数据，按时间列展开。

常见结构：

- 列 A/B：环号、测点编号
- 列 C：本次变量
- 列 D：盾构累计
- 列 E 以后：各时刻序列

常见分组：

- 上行线道床、下行线道床
- 上行线结构、下行线结构
- 上行线位移、下行线位移
- 上行线收敛、下行线收敛

### 4.2 `Z1 （静力水准)`

该页用于静力水准沉降序列，常见结构：

- A/B：环号、测点编号
- C：本次变量
- D：盾构累计
- E 以后：各时刻时间点

## 5. 计算与数值规则

- 全站仪平面变形按工程师坐标口径，不套高架桥墩倾斜公式。
- 沉降来源按用户模板和项目约定区分；若本项目明确“静力水准出沉降”，则沉降结论和静力水准页优先使用华桓静力水准 `curOffset/totalOffset` 或项目静力水准成果，不用全站仪平面成果替代。
- 水平位移以项目定义方向为准，通途西路模板中常见为：
  - 东西方向：`Y` 轴
  - 南北方向：`X` 轴
- 倾斜率只在高架桥墩模板里使用；本联合报表通常不出桥墩倾斜。
- 小数位一般按工程师模板：
  - 前台快报显示 0.1 mm
  - 数据页可保留更多小数
  - 静力水准页常保留 2 位或更高小数，按模板为准

## 6. 图表

工程师模板通常带有：

- `前台` 中的多张变化量曲线图
- `前台 (静力水准)` 的结构沉降曲线图
- `盾构总累计曲线图`
- `日变量曲线图`

要求：

1. 曲线图必须跟表内数据对应。
2. 曲线标题应反映上行线/下行线及监测项。
3. 若项目只提供单时段数据，也要保留辅助曲线页结构，不要删除图表逻辑。

## 7. 默认页面设置

- 纸张、方向、缩放和打印区域以工程师模板为准；不要因为脚本默认 A3 横向就覆盖用户最新模板设置。
- 封面页、联合快报页、静力水准页、辅助数据页要分别保持既定分页。
- 隐藏页只保留数据和图表计算，不作为用户直接阅读页。

## 8. 联合取数提示

当用户说：

- “通途西路这个模板”
- “前台要跟工程师的一样”
- “全站仪平面 + 静力水准沉降一起出”

优先提示其提供：

1. 全站仪平台 URL、项目 ID、平差网 ID。
2. 静力水准项目 ID 或华桓接口项目 ID。
3. 报表截止时间和上期时间。
4. 当前施工工况。
5. 点名映射和分区。
6. 左右示意图。
7. 初始值来源文件。

## 9. 数据来源分层

地下盾构穿越既有地下区间的联合报表必须先把数据源分层，不能把一个平台的成果硬套到所有监测项。

| 数据层 | 推荐来源 | 进入报表的位置 | 注意事项 |
|---|---|---|---|
| 项目信息和施工工况 | 用户输入、方案、工程师日报或现场记录 | `前台`、`前台 (静力水准)` 的工程概况、施工进度、结论 | 只作为文字和影响范围说明，不参与数值计算 |
| 全站仪平差成果 | 全站仪平台 `数据查询 / 平差与测站数据 / 变形监测平差数据`，由 `fetch_adjusted_total_station.py --project-template underground-shield` 导出 | `Z1 全 (2)`、必要时 `Z1 （正穿）`，再由 `前台` 汇总展示 | 只取已完成平差批次；`CZ` 为测站、`JD` 为测量基点，不作为监测点 |
| 静力水准沉降成果 | 上海华桓接口 `findSZByIdAndDate type=2`，由 `fetch_shhh_static_level.py` 导出 | `Z1 （静力水准)`、`前台 (静力水准)`，必要时回填 `前台` 的沉降类汇总 | `curOffset` 为本次变化，`totalOffset` 为累计变化；不要用原始液位或截图值替代 |
| 点名、线别、分区映射 | 布点图、方案、工程师确认表 | 全部数据页和前台页的点号、环号、上/下行线、影响区 | 必须人工确认；不要仅凭 `S/X` 前缀自动定稿 |
| 模板公式和图表 | 用户最新工程师 Excel 模板 | 最大值、时间序列、曲线图、打印区域 | 模板是展示模型，不是数据源；填数后应让公式和图表继续指向数据页 |

通途西路类项目的当前口径是“全站仪出平面变形，静力水准出沉降/高程变化”。遇到类似描述时，默认按下列来源优先级处理：

- 沉降/高程变化：优先用静力水准 `curOffset/totalOffset`；只有用户明确确认该项目沉降由全站仪承担时，才用全站仪 `H/Z` 坐标差。
- 水平位移：用全站仪平差后坐标差，方向按项目说明；常见为东西向看 `Y`，南北向看 `X`。
- 水平收敛：用全站仪成对点或模板既有公式；若点对规则不明确，先出点对确认表，不自动猜。
- 倾斜率：地下区间联合报表一般不填桥墩倾斜；除非项目另有高架桥墩监测项。

## 10. 数据落表映射

生成联合 workbook 时，不要直接在 `前台` 手填所有数值。推荐流程是“源 CSV -> 数据页/隐藏辅助页 -> 前台页/图表引用”。

### 10.1 全站仪 CSV 到 `Z1 全 (2)`

`fetch_adjusted_total_station.py` 输出的 CSV 至少需要这些字段：

- 时间：`previous_time`、`current_time`、`report_end`、`adjustment_time`。
- 点位：`point_id`、`ring_no`、`structure_zone`、`position_label`、`influence_zone`。
- 监测项：`monitoring_item`，常见为水平位移、水平收敛，若项目确认也可包含道床沉降、结构沉降。
- 数值：`current_change_mm`、`cumulative_mm`、必要时保留 `initial_x_m`、`initial_y_m`、`initial_h_m` 用于追溯。

落表规则：

1. 按模板既有分组顺序填入，不改变工程师模板的行组位置。
2. 每次出报在时间序列区追加一列当前时刻；不要覆盖历史列。
3. `C` 列本次变量优先由“当前时刻列 - 上期时刻列”或模板原公式计算；若脚本写值，必须与源 CSV 留痕一致。
4. `D` 列盾构/穿越累计按项目穿越起算时刻累计；`E` 以后为全历史或总累计序列，按模板标题为准。
5. 无效、缺测、遮挡、未平差点保留 `/` 或空值，不填 `0`。

### 10.2 静力水准 CSV 到 `Z1 （静力水准)`

`fetch_shhh_static_level.py` 输出的 CSV 至少需要这些字段：

- 时间：`previous_time`、`current_time`、`api_stat_date`、`api_ref_date`。
- 点位：`point_id`、`point_name`、`ring_no`、`structure_zone`、`location`。
- 数值：`current_change_mm` 对应接口 `curOffset`，`cumulative_mm` 对应接口 `totalOffset`。
- 追溯：`current_value`、`previous_value`、`current_original_value`、`previous_original_value`、`shhh_point_id`、`shhh_status`。

落表规则：

1. `A/B` 列按模板写环号和测点编号。
2. `C` 列写本次变化，`D` 列写穿越/盾构累计或模板指定累计口径。
3. 时间序列从 `E` 列以后追加当前时刻，图表跟随该页更新。
4. 线别、区段、影响区来自人工确认映射，不只依赖华桓平台点名。

### 10.3 前台页汇总

`前台` 和 `前台 (静力水准)` 只做展示汇总：

- 本次最大：在对应线别、监测项、有效点中按绝对值取最大；点号和值必须来自同一行。
- 穿越期间最大：按模板定义的穿越起算时刻到当前时刻的累计序列取最大。
- 总累计最大：若模板有总累计口径，按初始值到当前时刻的累计序列取最大。
- 并列最大时按工程师模板原有顺序或测点编号顺序取第一个，并在核对摘要中说明。
- `前台` 的沉降类项目若由静力水准承担，应从静力水准数据页汇总，不从全站仪 `垂直位移` 行重复取。

## 11. 展示方式和图表

工程师模板的展示逻辑优先于通用模板：

1. `前台` 保持上行线/下行线双列展示，但工程概况、施工工况、施工进度、结论与评价的文字口径应统一，不写两套互相矛盾的内容。
2. `前台 (静力水准)` 只展示沉降/高程变化，不出现水平位移、水平收敛或桥墩倾斜。
3. 曲线图必须引用数据页的时间序列，不要把图表截图粘贴到报表里替代可追溯数据。
4. 插入的工况图、影响范围图按模板图片框裁切填充，不越界、不压住标题和表格线。
5. 数值小数位按模板优先：前台常用 0.1 mm，数据页可保留 2-3 位或模板原精度；静力水准若工程师模板保留更多位，按模板执行。
6. 导出 PDF 前以打印区域为准检查分页，隐藏辅助页不应作为正式阅读页输出，除非用户明确要完整数据页。

## 12. 联合报表输入契约

后续要自动生成此类报表时，优先让用户或上游系统提供 `assets/underground-shield-combined-input-template.json` 同结构的配置文件。该配置文件不存账号密码，只描述：

- 报表名义时间、上期时间、出报频率。
- 全站仪和静力水准数据源路径或抓取参数。
- 施工工况、左右图片、点名映射和分区规则。
- 哪些监测项由全站仪负责，哪些由静力水准负责。
- workbook 中每个 sheet 的填报和展示规则。

如果该配置没有明确 `source_priority.settlement = static_level` 或 `total_station`，不得默认选择，应先让用户或工程师确认沉降来源。

## 13. 全站仪取数命令

通途西路类项目的全站仪点号常为 `S390`、`X400` 等环号/线别点，不是桥墩上下点。抓取全站仪平差成果时应增加 `--project-template underground-shield`，避免脚本把 `S390/X400` 误写成“390号桥墩/400号桥墩”。

```bash
python scripts/fetch_adjusted_total_station.py \
  --base-url "{{平台URL}}" \
  --prjid "{{项目ID}}" \
  --adjust-net-id "{{平差网ID}}" \
  --report-cadence "{{4h/2h/15min}}" \
  --report-cutoff-time "{{YYYY-MM-DD HH:mm:ss}}" \
  --previous-report-cutoff-time "{{YYYY-MM-DD HH:mm:ss}}" \
  --cumulative-source "coordinate-formula" \
  --current-change-source "previous-batch" \
  --project-template "underground-shield" \
  --work-condition "{{当前施工工况}}" \
  --output-dir "平台数据输出"
```

若再取静力水准数据，按 `crossing-static-level-report.md` 使用华桓 `findSZByIdAndDate`，再将两类 CSV 按工程师 workbook 的 `Z1 全 (2)` 和 `Z1 （静力水准)` 结构填入。

现有桥墩全站仪生成脚本会主动阻止此类联合模板误用：如果传入的模板包含 `前台`、`前台 (静力水准)`、`Z1 全 (2)`、`Z1 （静力水准)`，不要用桥墩生成器强行填报，应按本文件的联合报表流程处理。

## 14. 模板检查命令

拿到新的工程师 workbook 后，先检查结构再写生成逻辑：

```bash
python scripts/inspect_underground_shield_template.py \
  "{{工程师模板.xlsx}}" \
  --output "模板结构核对.md"
```

检查内容包括工作表显隐、尺寸、打印区域、图表数量、图片数量、合并单元格数量和关键公式。若模板缺少 `前台`、`前台 (静力水准)`、`Z1 全 (2)`、`Z1 （静力水准)` 中的关键页，应先向用户确认它是否仍属于本联合报表类型。
