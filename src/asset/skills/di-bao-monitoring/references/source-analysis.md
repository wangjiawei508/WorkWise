## 抽象出的通用流程

完整地保监测工作链可分为三段：

1. **实施前**：资料收集、监测方案、内审、专家评审、测点埋设、初始值采集、控制网第 1 期联测。
2. **实施中**：日报、周报、月报、巡视、控制网复测。
3. **事件处置和收尾**：预警/报警单、数据说明、专题会议、消警资料、停测资料、项目总结。

## 报表 Skill 必须去项目化

旧报表 Skill 中存在以下问题：

- 写入了样例项目全称。
- 写入了样例监测单位和建设单位。
- 写入了样例合同编号。
- 写入了固定首次监测日期。
- 写入了固定仪器编号。
- 写入了某一个项目的固定点数和点号范围。
- 监测项偏地下车站/区间，不够覆盖高架、桥墩、桥面场景。

新版 Skill 通过 `project_profile`、占位符模板、行级阈值覆盖和高架专项 reference 来避免这些问题。

## 报表对象差异

| 类型 | 常见结构 |
|---|---|
| 地下车站/盾构区间 | 上行线、下行线、道床沉降、结构沉降、水平位移、收敛、远端沉降、自动化/人工对比 |
| 高架/桥梁项目 | 桥下沉降/位移、桥墩沉降/位移/倾斜、桥面沉降、桥面道床沉降、桥面结构沉降 |
| 控制网资料 | 高程控制网、平面控制网、工作基点稳定性、成果沿用或更新 |
| 预警资料 | 级别、部位、监测项、点号、数值、阈值、时间、原因、措施、附件 |
| 初始值资料 | 测点埋设记录、测点验收记录、人工/自动化初始值、平面/高程控制网第 1 期联测 |
| 项目总结资料 | 目的依据、项目概况、人员设备、周期停测、工作量、控制网、成果分析、监护巡查、结论建议 |

## 日报 Excel 共性

多个项目日报工作簿通常包含：

- `info`、`监测日期`、`适配设置页`：项目元数据、日期、期号、天气、施工工况、报警状态、取数规则。
- `Z1`、`Z2`：自动化沉降和坐标类数据。
- `R1`、`R2`：人工沉降和坐标类数据。
- `累计数据（自动化）`、`累计数据（人工）`、`统计表`、`统计分析`：累计曲线、汇总和最大值。
- `封面`、`分析报告`、`前台`、各单项表页：报告输出页面。

因此，代理处理日报时应先读辅助表，再读输出页。

## 周报共性

周报通常比较短：

1. 封面或标题。
2. 本周监测工作量统计。
3. 外部工程/施工工况。
4. 监测数据汇总。
5. 存在问题及建议。
6. 下周工作计划。

不同项目的章节号可能略有差异，编辑既有报告时应保留原结构。

## 月报共性

月报通常包括：

1. 工程概况及进度。
2. 本月工作开展情况。
3. 控制网复测或控制网完成情况。
4. 监测数据汇总及分析。
5. 监测预、消警情况。
6. 存在问题及建议。
7. 巡视。
8. 附件。

月报不能只继承上一期工程概况，必须更新施工进度、工作量、数据、预警和附件。

## 初始值报告共性

初始值资料通常由以下部分组成：

- 测点埋设记录：测点编号、埋设时间、埋设说明、点位示意图、现场照片。
- 测点验收记录：测点数量、点号范围、对应监测对象、验收人员意见。
- 监测初始值审批表：人工监测初始值成果和自动化监测初始值成果分章。
- 平面控制网联测报告第 1 期：坐标系统、首级控制点、工作基点、网形、技术指标、成果更新原则。
- 高程控制网联测报告第 1 期：基准点、工作基点、水准路线、观测要求、平差高程、使用高程、成果更新原则。

## 项目总结共性

项目总结不是月报合订本，常见目录为：

- 目的及编制依据。
- 项目概况、位置关系、周边环境、施工节点。
- 投入人员及设备。
- 监测监护范围、周期及停测。
- 监测监护工作量、内容、精度、频率。
- 控制网建立与复测。
- 监测信息反馈和预警处置。
- 监测成果分析与评述。
- 分项节点变形分析。
- 监护巡查与安全生产总结。
- 结论与建议。

## 可复用规则

| 主题 | 规则 |
|---|---|
| 项目资料 | 先建资料卡，再写成果 |
| 期号 | 只能从当前项目上一期递推，不能套样例 |
| 日期 | 自动化和人工监测区间要分开 |
| 最大值 | 本次/本期最大和累计最大分别计算 |
| 报警字段 | `是否综合报警` 与 `是否数据报警` 分开 |
| 正负号 | 汇总表下方必须说明 |
| 控制网 | 月报应说明成果沿用或更新 |
| 初始值 | 正式监测前必须完成测点验收、初始值审批和控制网第 1 期联测 |
| 高架/桥梁 | 不默认写隧道收敛，优先按桥墩倾斜和桥面监测处理 |
| 项目总结 | 必须闭合监测周期、工作量、预警消警、停测依据和后续建议 |
| 模板 | 模板只保留结构，不保留样例项目固定内容 |
