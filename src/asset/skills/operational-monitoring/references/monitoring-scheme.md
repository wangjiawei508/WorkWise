# 监测实施方案的编制

监测实施方案（"监测方案"）是项目所有后续作业的总纲，先于测点布设、初始值采集、期次外业。本文档给出方案的章节骨架、必含要素和评审/修订规则。

## 1 方案的法律与商务地位

- **合同附件**：与业主签订的监测合同通常引用该方案，方案条款具有合同效力
- **专家评审**：方案在开工前须通过专家评审；评审意见与回复一并归档（见 `archive-and-delivery.md` §3）
- **变更控制**：方案变更需走"业主同意 + 监理签认 + 监测单位签发"流程，**不得**单方面调整
- **修订版本**：保留每个版本（V0.x 草稿 → V1.0 评审版 → V1.x 实施版），版本号在封面和页脚显式标注

## 2 方案章节骨架（共 5 篇）

### 第 1 篇：工作大纲与监测实施方案
1.1 **工程概况**
- 线路概况（起讫、长度、车站数、隧道形式 / 高架 / U 槽分布）
- 工程地质与水文条件（围岩等级、地下水位、特殊地质）
- 与运营本体、保护区周边环境的关系

1.2 **监测依据**
- 强制性国标 / 行标（GB/T 50308-2017、GB 50911-2013、GB/T 12897-2006、JGJ 8-2016、GB 50026-2007 等）
- 业主招标文件与技术要求
- 既有线设计、运营资料
- 专家评审意见（若已生成）

1.3 **监测目的**
- 长期监测、风险预警、积累资料、危害治理依据（详 `SKILL.md` §1）

1.4 **监测内容与方法**
- 内容：沉降 / 收敛 / 净空断面 / 水平位移 / 巡视等（详 `regulations-and-frequency.md` §2）
- 方法：水准 / 全站仪 / 激光测距 / 三维激光扫描 / 巡视

1.5 **工作量与监测频率**
- 各类点数（按区段、上下行、结构类型分别统计）
- 频率表（详 `regulations-and-frequency.md` §3）
- 工作筹划（人员排班、机械配合、运营天窗点对接）

1.6 **工作基点与监测点布设**
- 测点类型与实物（详 `monitoring-points.md` §1）
- 布设原则（盾构、车站、U 槽、联络通道、道岔、高架、建构筑物，详 `monitoring-points.md` §2）
- 点位命名规则（详 `monitoring-points.md` §4）
- **测点的保护与修复**（保护盒、复测、迁移流程）

1.7 **作业方法与技术要求**
- 监测基准网（详 `baseline-network.md`）
- 沉降监测（详 `settlement-monitoring.md`）
- 收敛与三维激光（详 `convergence-monitoring.md`）
- 水平位移（详 `horizontal-displacement.md`）
- 巡视巡检

1.8 **变形控制标准**
- 各监测项目的预警值 / 控制值 / 速率限值
- 工后变形限值（一般 ≤15 mm；敏感段 ≤10 mm；高架差异沉降 ≤2 mm/m）
- 与项目方案/合同条款的对应（详 `warning-and-disposal.md` §2）

1.9 **监测点信息化管理、保护与修复**
- 点位台账（点号、坐标、布设日期、责任人）
- 保护盒/标识/警示
- 损坏与迁移流程（原点号映射表）

### 第 2 篇：技术管理与质量控制
2.1 **项目组织架构**：项目负责人、技术负责人、外业组、内业组、QC、安全员
2.2 **人员配备**：姓名、职称、资格、与本项目分工
2.3 **仪器设备配备**：型号、数量、精度、检定有效期
2.4 **技术管理措施**：交底、复核、内审、外审
2.5 **质量方针、目标、保证体系、控制措施**
2.6 **职业健康与安全保障措施**：天窗点纪律、轨行区准入、接触网作业、PPE、应急救援
2.7 **环境保护措施**：噪声、扬尘、废弃物

### 第 3 篇：信息化监测与成果反馈
3.1 **数据传输链路**：外业 → 内业 → 业主
3.2 **数据存储与备份**：原始数据 + 平差工程 + 报告，双备份
3.3 **成果交付方式**：日报 / 周报 / 月报 / 期报 / 总结报告
3.4 **预警反馈机制**：黄/橙/红三级，4 小时内快报（详 `warning-and-disposal.md`）

### 第 4 篇：项目重难点分析与对策
4.1 **项目认识**：本项目独有的风险点（如保护区基坑密集、敏感段、特殊结构）
4.2 **重点难点**：典型如——
- 短天窗点高效作业
- 三维激光扫描在带电环境下的安全实施
- 联络通道与正线衔接段数据连续性
- 类矩形 / 矩形盾构非常规断面建模
- 长期复测点位损坏率控制
4.3 **应对措施**：每个难点对应技术/管理对策
4.4 **应急预案**：
- 预警/报警/紧急触发后的响应流程
- 加密观测启动
- 与业主/运营/应急组联动
- 人员/仪器后备方案
- 通信链路冗余

### 第 5 篇：附录
- 既有控制网成果（业主提供）
- 测点平面布置图（按线分册）
- 仪器检定证书清单
- 人员证书清单
- 合同副本 / 招标文件相关条款摘录
- 评审意见与回复
- 方案修订对照表（按版本）

## 3 编制流程

```
1) 招标/中标后启动
   │
2) 收集基础资料（既有线设计、运营资料、地勘、控制网成果、招标文件）
   │
3) 现场踏勘（识别保护区、敏感段、施工干扰）
   │
4) 起草初稿（依本骨架）→ 内审 V0.x
   │
5) 提交业主 → 形式审查
   │
6) 专家评审 → 评审意见
   │
7) 修订 → V1.0 评审版
   │
8) 业主与监理签认 → V1.0 实施版
   │
9) 全员技术交底（外业 + 内业）
   │
10) 正式实施；过程中需变更走变更流程
```

## 4 评审通过要点（专家最常关注）

- **依据完整**：所有强制性规范列全、与监测内容对应
- **限差明确**：所有等级（一等/Ⅱ级/水平位移Ⅱ级）限差表完整无歧义
- **频率合理**：与运营阶段、保护区、敏感段差异化设置
- **预警值有出处**：来自规范、设计、合同的具体条款
- **应急预案可执行**：联系人电话、响应时限、加密启动条件、消警条件均量化
- **测点保护到位**：损坏率预估与修复机制
- **工后变形零点定义清晰**：永久不变的期次基准
- **质量保证体系闭环**：内审 + 外审 + 业主签收均有承诺时限
- **重难点不空话**：每一条难点对应一条对策，对策可被检验

## 6 方案变更控制

### 6.1 触发条件
- 监测对象变化（增加/撤除区段、保护区新基坑接入）
- 频率调整（业主同意降频或升频）
- 仪器升级（影响精度评定基准）
- 限差/预警值改动
- 人员重大变动

### 6.2 变更流程
```
监测单位发起【变更申请单】
   → 监理审查
   → 业主审批
   → 监测单位发布变更附件（V1.x）
   → 全员技术交底
```

变更不溯及既往：变更前的期次成果按旧方案口径成立。

## 7 常见 Pitfall

- 把"招标文件原文"复制为方案核心条款，未做项目化
- 没有专家评审就先开工
- 测点保护章节空泛，运营 1 年后损坏率 >10% 无应对
- 工后变形零点未在方案中固化，跨年度被业主追溯
- 应急预案只写"4 小时内上报"，未列联系人和决策权限
- 频率表与合同条款不一致
- 变更走"口头同意"不立单，年终审计无法追溯
- 与"运营天窗点"的衔接只写一句，未明确班次、协调方式
