# WorkWise 0.2.9

0.2.9 是一次以真实用户场景为准的交付质量修复版本。我们由产品经理、前端体验、软件测评和一线工程师四类角色完成了独立候选版测试，重点修复 PPT、文档交付、任务完成、SSE、首次启动和 CLI/Skill 可发现性问题。

## PPT Master 与成果交付

- 内置 PPT Master 更新到 `3.1.0+` 基线，并在插件市场显示清晰的标准名称和版本。
- 写作助手选择 PPT Master 后可在受控工作区内读取技能并调用专用导出工具，最终必须交付真实 `.pptx`；HTML 不再作为 PPT 的替代成果。
- 成果文件卡的“另存为”“在编辑器中打开”“显示位置”均使用文件所属工作区进行校验，失败会显示明确原因，不再出现按钮无反应。
- 修复窄窗口下成果文件名和文件类型难以识别的问题。

## 任务完成与连接稳定性

- 修复阶段性回复被误判为完成、任务中途停住以及重复补救导致重复消息的问题。
- 完成守卫只读取真实用户请求，正确识别“不要生成文件”等否定要求，避免无意义地反复要求交付文件。
- 修复 SSE 停止请求早于启动完成时产生孤立连接的竞态；会话切换、取消和退出后会及时回收连接。
- 取消后的 Turn 只写入一个终态，输入框和后续发送能力可正常恢复。

## 写作与文档导出

- Markdown 的 Word 导出改为真实 OOXML `.docx`，保留中文、标题层级、项目符号、编号、表格、链接、代码和本地图片。
- 移除实质为 HTML 改扩展名的旧 `.doc` 入口；导出菜单只保留 HTML、PDF 和 Word 文档（DOCX）。
- 全新中文用户首次启动后自动生成中文欢迎文档；语言切换只更新未修改的默认欢迎页，不覆盖用户内容。
- 无模型密钥时仍可直接进入本地写作；中号/大号界面字体能够保存并在重启后保持。

## CLI、Skill 与知识库

- 三个 CLI 始终显示；GitHub 下载增加超时、瞬时网络错误重试和可诊断失败信息。
- 可信内置 Skill 与网络安装包继续使用不同的安全边界，避免模板参考文件被错误当作超大网络包。
- RailWise KB 检索会注入公开索引、真实条目和来源链接，不再回答“没有知识库”。

## 验证与发布范围

- 967 项 WorkWise 自动化测试通过，1 项按平台条件跳过；内置运行时 538 项测试通过。
- TypeScript、ESLint、品牌边界、OpenSpec strict、生产构建和构建新鲜度检查通过。
- macOS Apple Silicon 候选包完成 ASAR 一致性、WPS DOCX 实机打开和多角色界面复测。
- GitHub Release 公开区域只提供三个客户端：macOS Apple Silicon、macOS Intel 和 Windows x64。

## 升级说明

- 建议 0.2.8 用户升级到 0.2.9。
- 如果旧版本已出现 SSE 连接上限提示，请先使用 `Command + Q` 完全退出 WorkWise，再安装新版本。
- 现有工作区、会话、设置、CLI 和 Skill 数据保持不变。
