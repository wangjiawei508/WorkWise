# WorkWise 0.2.8

0.2.8 是针对 0.2.7 用户反馈的稳定性修复版本，重点解决任务提前结束、成果交付、界面设置、写作工具、CLI/Skill 市场、微信响应和 SSE 连接泄漏问题。

## 任务完成与成果交付

- 增加任务完成守卫：智能体在目标尚未完成、仍有待办项或尚未交付成果时，不再把普通阶段性回复误判为任务完成。
- 对可继续执行的中间状态自动推进，并对重复停顿设置明确边界，减少需要用户反复输入“继续”的情况。
- 文档、表格、演示文稿等成果在会话中显示为可交付文件，提供打开文件、在文件夹中显示和另存/下载入口。
- 修复超长任务中 SSE 恢复后可能丢失终态或重复结束的问题。

## 界面与写作工具

- 修复“设置 → 通用”中界面字体选择中号或大号后提示应用失败、返回后恢复小号的问题。
- 修复写作台未打开文件时“生成配图”和“生成 PPT”按钮不可用的问题；生成配图会先创建草稿，PPT 入口会正确准备提示词。
- 补齐写作台 PPT 按钮中文文案，不再显示 `writeGeneratePpt` 等内部键名。

## CLI、Skill 与手机连接

- CLI 市场始终展示飞书命令行工具、Office 文档工具和 Ego 浏览器助手；已安装但需要登录或诊断异常的工具不再从列表中消失。
- 修复 Codex 官方模板插件中的 `reference.pptx` 被错误套用网络安装 1 MiB 限制的问题。
- 网络安装 Skill 仍保持严格的文件数量、单文件大小、总量、链接和路径安全检查；可信插件只使用受控的只读发现规则。
- 微信消息发送增加重试、处理中反馈和失败提示，避免长时间无响应且用户无法判断状态。

## SSE 连接稳定性

- 修复停止或切换会话后 SSE 连接计数未立即释放，最终导致所有会话提示“窗口 SSE 连接数已达上限”的问题。
- 同一线程重新订阅时自动回收旧连接，不同子会话仍可并行接收事件。
- 窗口关闭、取消操作和应用退出时统一清理 SSE；达到资源上限时写入可诊断日志。

## 验证

- 948 项自动化测试通过，1 项按平台条件跳过。
- TypeScript、ESLint、品牌边界检查和生产构建通过。
- GitHub Release 公开区域只提供三个客户端：macOS Apple Silicon、macOS Intel 和 Windows x64。

## 升级说明

- 建议所有 0.2.7 用户升级到 0.2.8。
- 如果 0.2.7 已出现 SSE 连接上限提示，在安装新版本前可先使用 `Command + Q` 完全退出 WorkWise 后重新打开；升级到 0.2.8 后不应再次累积。
- 现有工作区、会话、设置、CLI 和 Skill 数据保持不变。
