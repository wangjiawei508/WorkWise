# WorkWise 软件介绍

> 让 AI 进入真实工作流。

WorkWise 是一款本地优先的桌面 AI 工作台。它把项目文件、会话、文档写作、可复用技能和外部工具组织在一个应用中，帮助个人与团队完成需要持续上下文、反复迭代和正式交付的工作。

[产品主页](https://www.railwise.cn/products/workwise/) · [使用指南](./USER_GUIDE.zh-CN.md) · [版本与下载](https://github.com/wangjiawei508/WorkWise/releases) · [问题反馈](https://github.com/wangjiawei508/WorkWise/issues)

## 产品定位

普通 AI 对话擅长回答一次性问题，但真实工作通常还包含资料目录、多个文件、修改历史、审批边界、团队规范和最终交付格式。WorkWise 的目标是把这些要素放回同一个工作现场，让 AI 在可查看、可中断、可复核的边界内持续协作。

它适合以下用户：

- 需要 AI 理解和修改本地项目的开发者与技术团队。
- 需要编写方案、报告、标书、汇报材料和知识文档的专业人员。
- 希望把个人方法、团队规范和行业模板沉淀为可复用 Skills 的组织。
- 需要在本地工作区内连接模型、工具和公开知识来源的高级用户。

## 主要工作方式

### 1. Code 工作台

Code 围绕本地项目目录工作，可用于理解代码或资料、拆解需求、修改文件、执行工具、运行测试和审查结果。计划、Todo、审批、工具输出和文件差异保留在同一任务上下文中，便于追踪过程和回退判断。

适合：

- 功能开发、缺陷修复与代码审查。
- 项目资料整理和批量文件处理。
- 需要多步执行、检查和验证的本地自动化。
- 将模糊需求整理为可执行计划。

### 2. Write 写作工作台

Write 面向文档型工作，提供 Markdown 编辑、实时预览、选区助手、文件级会话和导出能力。发送给写作助手前会先保存当前文档，知识检索结果以来源链接进入内容，便于人工核查。

适合：

- 技术方案、项目报告、周报和总结。
- 文章、说明书、知识库内容与产品文档。
- HTML、PDF、DOC、DOCX 等格式的交付准备。
- 借助 PPT Master 等 Skills 规划演示文稿。

### 3. Skills

Skill 是一组可复用的工作说明、参考资料和脚本。它可以表达团队写作规范、行业流程、测试方法或交付模板。WorkWise 会在安装前检查包内路径、文件数量、体积和异常文件类型，避免不安全内容越过允许边界。

### 4. MCP 与命令行工具

MCP 用于连接外部工具和数据源；命令行工具由 WorkWise 安装到独立目录，避免修改系统全局环境。需要独立应用的能力会明确标注并打开官方安装页，不会伪装成已经完成安装。

### 5. 知识检索

Write 可将本地资料检索与 RailWise 官方知识库结合。远程查询只发送压缩后的关键词，异常或离线时回退到公开静态知识内容和本地缓存。涉及标准、阈值、频率和正式结论时，仍需由专业人员复核。

## 产品原则

### 本地优先

工作区、会话、设置和多数缓存以本机为中心。WorkWise 不要求把整个项目上传到自有云端；模型请求的实际数据边界取决于用户选择的模型服务、提示内容和组织策略。

### 过程按需可见

默认界面只展示对话、短进度、审批、异常和成果，避免脚本、工具参数和长输出淹没用户目标。需要排查时可以打开“工作详情”，切换到标准或开发者模式查看脱敏后的语义操作、命令摘要、Diff 和指标。模型私有思维链在任何模式下都不会显示。

### 任务必须经过验收

0.3.0 以持久化任务而不是单个模型 Turn 作为完成单位。普通问答必须有最终回复；文件任务必须有真实、可操作的成果；Office/PDF 成果必须通过结构验证。步骤触顶或暂时异常会保存 checkpoint 并继续，只有审批、必要输入、安全阻断、预算耗尽、不可恢复错误或主动取消才暂停。

### 文档能力本地优先

WorkWise 内置经过固定版本审计的 Microsoft MarkItDown sidecar，用于 PDF、DOCX、PPTX 和 XLSX 的本地结构化解析；PDF 页面显示由 PDF.js 完成。扫描件、复杂多栏、公式或跨页表格可选择安装 MinerU 3.4 系列本地增强组件。MinerU 不随客户端分发，文档也不会被自动上传到 MinerU 公共云端。

### 能力有边界

稳定功能、预览功能和后续方向会明确区分。第三方 Skill、CLI 和项目必须展示来源；没有清晰许可证的上游内容只提供外部入口，不随 WorkWise 分发。

### 结果需复核

AI 生成内容可能存在事实、计算、引用和排版错误。涉及工程安全、合同、财务、法律、医疗或正式对外文件时，WorkWise 不能替代具备相应责任和资质的人员审核。

## 更新、语言与帮助

- 中文和英文界面由同一语言设置驱动，包括 macOS 原生菜单、Windows 应用菜单和托盘菜单。
- WorkWise 默认从官方 GitHub Releases 检查更新；发现新版本时显示提醒。
- 用户可在“帮助 → 检查更新”或“设置 → 通用 → 软件更新”主动检查。
- 帮助菜单提供产品主页、个人主页、软件介绍、GitHub 项目、版本下载和日志目录。

## 安装与系统要求

公开桌面安装包包括：

- macOS Apple Silicon。
- macOS Intel。
- Windows x64。

当前不提供 Linux 桌面客户端、便携版或激活码。模型 API Key、可用模型、额度和计费由用户选择的服务商及账户权限决定。

MarkItDown 已包含在三种客户端中，不要求用户安装 Python。MinerU 属于可选组件，安装前会检查平台、内存和磁盘空间；普通用户无需安装即可使用基础 PDF/Office 解析和预览。

## 数据与隐私提示

1. 只在获得授权的工作区中运行 WorkWise。
2. 不要在截图、Issue 或日志中公开 API Key、客户资料和内部路径。
3. 安装第三方扩展前核对项目来源、许可证、发布资产和权限范围。
4. 处理敏感信息时优先使用组织批准的模型服务和网络环境。
5. 正式交付前检查事实、引用、数值、图片、表格和排版。
