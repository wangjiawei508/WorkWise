## 真实应用回归

测试机：

```text
macOS 26.5.2 (25F84)
Apple Silicon arm64
```

重新构建的未签名 Apple Silicon 候选应用可由 Codex Computer Use 正常启动和操作，未覆盖 `/Applications/WorkWise.app`。真实界面完成以下回归：

1. 打开已保存的 Design 文档 `0.3.1 候选版完整测试`，确认画板、文字、图形和保存状态可恢复。
2. Design 助手执行“修改文字并新增图形”，Runtime 调用 `design_apply_canvas_commands`，画板显示新文字和新图形，并显示“已将 2 项修改应用到当前画板”。
3. 撤销恢复原文字和图形状态；重做再次恢复助手修改。
4. 正常退出应用后，WorkWise 主进程、Agent Runtime 和 Schedule 辅助进程均在 1 秒内退出；重新启动后最新画板内容仍存在。
5. Design 工具栏中的导出 PPTX、下载 PNG、下载 SVG、插入 Write 操作均处于可操作状态。此前同候选代码已完成四类成果的真实文件生成、成果卡打开/另存为/显示位置，以及 Write 插入后清理。
6. 最终依赖锁和 Runtime 启动环境修正后重新打包并再次真实启动，确认版本为 `0.3.1`、Design 文档和最新画板内容恢复、四个交付入口可用；从中文应用菜单退出后，主进程、Runtime 和 Schedule 辅助进程全部退出。

回归过程中发现端口 `8899` 曾被 2026-07-20 的旧测试包 Runtime 占用。旧健康响应与新 Runtime 的基础字段相同，导致新 UI 误连旧进程，并出现 `unknown tool: design_apply_canvas_commands`。修复后：

- `/health` 增加固定 `protocolVersion`。
- 主程序只接受与当前客户端协议版本完全一致的 Runtime。
- 无协议字段的旧 Runtime 不再被静默当成兼容服务。
- 新增主程序和 Runtime 健康检查回归测试；相关 20 项定向测试全部通过。
