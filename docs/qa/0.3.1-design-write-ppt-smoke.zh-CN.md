## 2026-07-24 Design 紧急回归

用户在最终候选之后发现两个真实阻断缺陷：

1. Design 画布继承了应用外壳的 `-webkit-app-region: drag`，拖动画布会拖动整个桌面窗口，画布自身没有可靠的平移交互。
2. Agent 常用的 SVG 风格参数（`path`、嵌套 `style`、带 `#` 的颜色、`fill: none`、圆角端点/连接）没有被完整接受和持久化；模型重试时还可能消费过期命令，最终显示“Active Design document changed”但画布为空。

修复后的本机开发应用完成了以下真实界面回归：

- Design 视图和画布明确取消桌面拖动区域，仅保留顶部标题栏作为窗口拖动区。
- 拖动灰色空白舞台后，画布从约 `(322, 132)` 移动到 `(376, 169)`，桌面窗口位置不变。
- 缩放按钮可将画布比例从 `53%` 调整到 `64%`，适合画布可恢复居中视图。
- 新建 `Logo 最终回归` 文档并发送“画一个简洁的蓝绿色字母 R Logo，放在画布中央”，画板真实显示居中的蓝绿色 R，助手提示“已将 3 项修改应用到当前画板”，没有过期文档错误。
- 保存的 `.workwise-design.json` 包含 1 个蓝绿色矩形和 2 个蓝绿色路径，路径保留 `strokeWidth: 60`、`strokeLinecap: round` 和曲线路径的 `strokeLinejoin: round`。
- 切换到其他 Design 文档再返回后，R Logo 和成功状态仍然存在，证明成果不是瞬时预览。
- 命令消费改为只检查最新用户请求之后的工具结果，只接受该请求中第一个匹配当前工作区、文档和页面的原子命令；同一请求的模型重试被静默忽略。

这次修复新增画布平移/缩放、命令请求隔离、SVG 风格归一化、路径圆角渲染/导出和持久化回归测试。紧急修复完成后重新执行并通过：
