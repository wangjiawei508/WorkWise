## 问题

### P2-01 首次配置可在没有 Key 时直接关闭，随后进入大面积不可用状态

- 复现：全新隔离 profile 启动 → 不输入 Key → 点击“关闭”。
- 预期：明确提供“离线写作模式”选择并说明可用范围，或把“以后配置”与“关闭窗口”区分开。
- 实际：用户进入 Code 工作台后，新建会话、新建需求和输入框均不可用，只看到运行时唤醒/重试提示。
- 证据：`01-first-run-zh.png`、`02-workbench-runtime-blocked.png`。

### P2-02 供应商页两个选择器的默认值互相冲突

- 复现：无 Key 状态打开设置。
- 预期：预设选择器显示“请选择预设”或与当前供应商 DeepSeek 一致，并解释“预设”只用于添加配置。
- 实际：上方“供应商预设”为 Xiaomi，下方当前供应商为 DeepSeek，首次用户容易误以为配置错位。
- 证据：`03-settings-provider-gate.png`。

### P2-03 简体中文界面的默认 Welcome 文档仍为英文

- 复现：简体中文界面进入 Write → 打开 `welcome.md`。
- 预期：默认示例文档随界面语言本地化，或提供中英双语模板。
- 实际：文档标题、正文和列表全部英文，破坏首次使用的一致性。
- 证据：`10-write-export-options.png`。

### P3-01 帮助文案把 Skill 安装入口写成“设置的 Skills / 外部工具页面”

- 复现：设置 → 帮助 → 插件市场与 Skills。
- 预期：描述当前真实路径“左侧插件 → 技能/命令行工具”。
- 实际：文案指向设置内页面，与现有信息架构不一致。
- 证据：`05-help-center.png`。

### P3-02 运行时离线横幅占用高优先级视觉，但未直接给出“继续本地写作”动作

- 复现：无 Key 进入 Write。
- 预期：除重试/设置外，提供“继续本地写作”或一句明确的离线可用能力提示。
- 实际：横幅以错误/暂停语气呈现，虽然本地编辑实际可用，用户仍容易认为整个 Write 失效。
- 证据：`09-write-empty-state.png`、`10-write-export-options.png`。

## 发布阻断项

1. 核心 AI 任务、自动继续、SSE 稳定性和 Agent 成果卡下载/显示位置无法在无 Key 的隔离 profile 中完成；必须由有测试凭据的 QA/工程角色补齐后才可宣称全面通过。
2. P2-03（中文默认文档仍为英文）建议作为中文体验发布阻断项修复。
3. P2-01、P2-02 至少需产品确认是否接受；若目标是“安装后简单可用”，建议发布前修复。
