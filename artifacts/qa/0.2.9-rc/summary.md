## 已修复并回归的问题

1. 简体中文首次启动选择“先使用本地写作”后，语言、中文菜单和中文 `welcome.md` 正确持久化。
2. 无 API Key 时可直接进入本地 Write，不再进入不可用的 Code 工作台。
3. 中/大字体可应用、保存并在重启后保持；窄窗口下侧栏自动转浮层。
4. CLI 三项同时显示；PPT Master 使用标准名称并展示 `3.1.0+` 基线。
5. Write 的生成图片与 PPT 入口恢复可用；PPT 任务必须交付真实 `.pptx`，不再用 HTML 冒充。
6. 成果卡“另存为 / 在编辑器中打开 / 显示位置”三项入口已逐文件真实验证，失败不再静默。
7. SSE 的“停止先于启动完成”竞态已修复；连续切换 12 个会话无连接上限、重复完成事件或悬挂线程。
8. 任务取消写入唯一 `aborted` 终态，输入和发送能力正常恢复。
9. “不要生成文件”不再被完成守卫误判为缺少文件交付；真实 AgentLoop 集成测试确认只调用模型一次、只产生一条回复和一次完成事件。
10. RailWise KB 注入公开索引、真实条目和来源链接，不再回答“没有知识库”。
11. 日志和用户可见文案使用 WorkWise Runtime；生产构建增加新鲜度校验，避免发布旧 `out/`。
12. 旧 DOC 导出确认为 HTML 改扩展名，已从菜单隐藏；Word 导出统一为真实 OOXML `.docx`，保留标题、列表、表格、链接、代码和本地图片。

## WPS 与首次启动实机复测

以下项目已在本机候选版和 WPS Office 中通过：

1. 导出菜单只显示 HTML、PDF、Word 文档（DOCX），不再显示旧 DOC。
2. 导出的 `.docx` 在 WPS 中正常显示中文、标题层级、项目符号和表格。
3. DOCX 是真实 OOXML 文档，可继续编辑且未出现 HTML 兼容格式警告。
4. 全新隔离用户目录首次启动后，设置语言为中文，默认 `welcome.md` 同步生成中文内容。
5. 用户修改过的欢迎文件不会被语言切换覆盖。
